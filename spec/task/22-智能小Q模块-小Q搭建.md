# 第6点：智能小Q模块 - 小Q搭建

本点覆盖的官方页面主要有 3 类：`小Q搭建概述`、`小Q搭建操作说明`、`指令支持列表`。这 3 组页面合起来，才能看清小Q搭建不是单一“AI 生成图表”功能，而是一套围绕仪表板搭建、图表配置、样式美化和报表操作的对话式 Copilot 能力。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)

## 点位定义
`小Q搭建` 是 Quick BI 在智能小Q体系里的“报表生产层 Copilot”。它面向的是仪表板搭建过程本身，不是单纯帮用户查数或写报告，而是通过自然语言直接完成图表创建、字段调整、分析元素添加、样式美化、发布分享等一整套仪表板生产动作。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)

## 这个点解决什么业务问题
- 解决“搭报表太依赖专业分析师”的问题，让用户通过自然语言完成原本需要手工拖字段、调样式、设分析项的工作。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 解决“仪表板制作效率低”的问题，支持基于数据集一键生成图表或整张报表，明显缩短从数据到可视化结果的时间。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 解决“样式美化依赖设计经验”的问题，支持一键智能美化、品牌 Logo 参考美化和组件配色调整，让报表更快达到可展示状态。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 解决“分析动作和搭建动作割裂”的问题，在搭报表的同时就能加入同环比、辅助线、趋势线、异常检测、摘要生成等分析能力。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)

## 核心功能能力
- 对话式创建图表：输入数据集名称和分析需求后，系统可自动生成图表，并支持继续通过对话修改图表类型和标题。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 一键生成报表：不仅能建单张图，还能基于已保存数据集快速生成整张仪表板，并支持重新生成或撤销。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 图表配置自动化：支持修改字段、改字段展示名、补字段描述、加后缀、调聚合方式、排序和数据格式等，等于把大量图表配置动作转成自然语言操作。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 分析元素增强：支持添加同环比、总计、小计、目标值、条件格式、辅助线、趋势线、最值、标注、自动联动等，说明它不是只负责“画图”，还负责把分析组件补齐。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 查询控件搭建：支持按报表级、Tab 级、图表级添加查询控件，并支持查询控件置顶。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 智能洞察辅助：可直接生成报表摘要、查看同环比波动原因，并发起异常检测，把“搭建”与“洞察”结合起来。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 一键智能美化：支持内置风格美化、基于企业 Logo 参考品牌色美化，以及单图表配色调整。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 报表级操作：支持保存、发布、分享、切换编辑态/预览态、水印开关等，让 AI 从“搭建助手”延伸到“交付助手”。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 多轮对话理解：支持基于上下文连续理解意图；若开启新会话，则同步清空上下文记忆。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)

## 典型使用场景
- 新建经营看板：基于一个数据集快速生成整张销售、运营、财务类报表。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 现有报表优化：已有人做出初版图表后，用自然语言继续调字段、换图型、加查询控件和分析元素。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 会前快速出稿：在时间紧张场景下，先一键生成报表，再一键美化与补摘要，快速形成可汇报版本。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 品牌化对外展示：通过品牌 Logo 或主题风格统一页面视觉，用于客户展示、经营汇报等正式场景。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)

## 适用角色
- 数据分析师：能大幅减少仪表板制作和调优的重复劳动。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 业务分析人员：即使不是重度 BI 设计者，也能较低门槛搭出可用报表。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 售前/经营汇报人员：适合快速生成展示型看板和专题页面。
- 组织管理员：负责购买模块并分配席位。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)

## 关键使用路径
- 第一步：购买 `小Q搭建 Agent`，并由组织管理员分配席位。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 第二步：进入 `新建仪表板` 页面，从右上角打开小Q搭建对话框。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 第三步：选择数据集，直接输入建图或建报表需求，或者进入指令中心按模板发起操作。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 第四步：围绕已生成图表继续做字段、样式、分析项、查询控件等配置。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 第五步：生成摘要、做异常检测、完成美化，然后保存、发布或分享仪表板。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)

## 前置条件 / 权限要求 / 版本限制
- 小Q搭建属于增值模块，需要额外购买；购买后还需分配席位，未分配席位的成员无法使用。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 仅 `高级版` 和 `专业版` 支持，`个人版` 不支持。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 当前官方明确写明：仅 `中国香港` 和 `马来西亚` 站点支持该模块，其他站点陆续开放中。这是非常关键的版本/地域限制。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 功能入口在 `新建仪表板` 页面，说明它目前核心作用域是仪表板生产流程，而不是全站任意页面。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)

## 与上下游点位的关系
- 上游依赖：它承接的是数据集、仪表板编辑能力和智能小Q Agent 席位。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[Quick BI 智能小Q概述](https://help.aliyun.com/zh/quick-bi/user-guide/smartq)
- 与前面的 `小Q问数`、`小Q解读`、`小Q洞察`、`小Q报告` 不同，小Q搭建更偏“生产报表”，而不是“消费数据”或“输出文档”。这是它在智能小Q体系里的独特位置。
- 下游承接：搭好的仪表板可以继续被 `小Q解读`、`小Q问数`、`小Q报告` 等能力消费，因此它更像智能 BI 链路里的“内容生产入口”。

## 限制与边界
- 官方明确说明：相关指令仅支持在 `仪表板模块` 中执行，不是通用于所有 Quick BI 页面。[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 图表级操作前，需要先选中目标图表，再向小Q发送指令，否则容易操作不到正确对象。[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 并不是所有组件都支持所有指令，官方对不同能力都给了具体支持范围。例如趋势线、辅助线、条件格式、组件配色等都各自限定了支持图表类型。[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
- 小Q搭建虽然包含“智能洞察归因”，但它这里的定位更偏报表内辅助分析，不等同于后续独立的 `小Q洞察` 经营洞察体系。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)、[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)
- 多轮对话会保留上下文；如果开启新会话，上下文记忆会被清空，因此长链路搭建要注意会话连续性。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)

## 售前 / 方案视角价值解读
- 小Q搭建最适合被讲成“AI 报表搭建 Copilot”，不是只会回答问题的聊天助手，而是直接参与 BI 内容生产。
- 对客户价值很直接：降低仪表板制作门槛、缩短交付时间、提升展示效果，尤其适合分析团队人手有限但报表需求很多的场景。[小Q搭建概述](https://help.aliyun.com/zh/quick-bi/user-guide/intelligent-assistant-of-quick-bi)
- 对售前演示来说，小Q搭建很适合做“现场搭报表”演示，因为它能在短时间内呈现从数据集到图表、到美化、到发布的完整链路。
- 对方案落地来说，它的真正价值不只是“快”，还在于把大量原本散落在图表配置、分析增强、视觉风格调整里的细碎动作收敛成统一自然语言接口，这对推广 BI 自助化很有帮助。
- 但也要提前讲清边界：它当前明显更适合仪表板场景，而且受地域站点和组件支持范围限制，不能按“全场景万能 AI 设计器”去承诺。[小Q搭建操作说明](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-bi-copilot)、[指令支持列表](https://help.aliyun.com/zh/quick-bi/user-guide/supported-directives)
