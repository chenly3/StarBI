# 第9点：系统管理 - 账号管理

这一点只覆盖 `系统管理-账号管理`。
它和后面的 `权限管理` 要分开看：这一点解决的是“用户怎么进入 Quick BI 组织、怎么被归类、怎么加入工作空间、怎么被维护和停用”；后面的 `权限管理` 才会深入讲“这些用户具体能看什么、能操作什么、如何集中授权”。官方在成员管理概述里把账号定义为“在工作空间中使用的主体”，已经把这一点的核心边界说得很清楚。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)

## 点位定义
`账号管理` 在 Quick BI 里本质上是“组织身份与成员运营层”。
它不是简单的“开账号”，而是一整套 `账号接入 -> 用户类型划分 -> 组织角色绑定 -> 工作空间加入 -> 用户组归类 -> 工作空间模式协作 -> 账号维护与停用 -> 个人账号绑定与登录管理` 体系。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)

## 这个点解决什么业务问题

- 把企业里的不同身份主体接入 Quick BI，形成统一的组织用户体系，而不是靠零散个人注册。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 在组织层面对用户做基础分类，区分 `开发者 / 分析师 / 访问者` 三类使用者，避免“一种账号打天下”。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 把用户进一步纳入 `组织角色、空间角色、用户组、工作空间` 这些组织结构中，形成企业级协作框架。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members) [创建用户组](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-user-group)
- 让管理员可以对用户进行查询、批量更新、导出、停用、删除和交接，形成可维护的账号生命周期管理。[管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)
- 让个人用户可以进行设备窗口管理、账号绑定和登录规则管理，降低混用账号与登录安全问题。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 核心功能能力

- `统一账号接入`：Quick BI 支持接入 `阿里云主账号、阿里云 RAM 账号、阿里云 RAM 角色、钉钉账号、企业微信账号、飞书账号、Teams 账号`。添加方式支持 `手动添加、批量导入、RAM 自动读取、从办公软件同步`。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- `用户类型体系`：组织层有三种用户类型：
  - `开发者`：偏 IT、数据分析和数据运营人员。
  - `分析师`：偏业务分析人员，可创建仪表板、电子表格等。
  - `访问者`：偏只读消费用户，只看报表、站点和订阅结果。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- `组织角色体系`：预置 `组织管理员、权限管理员、普通用户`，专业版还支持自定义组织角色。这里解决的是组织级管理职责，而不是具体资源可见范围。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [角色管理](https://help.aliyun.com/zh/quick-bi/user-guide/role-management)
- `空间角色体系`：预置 `空间管理员、空间开发者、空间分析师、空间查看者`，专业版支持自定义空间角色。它解决的是用户进入某个工作空间后，扮演什么协作身份。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [角色管理](https://help.aliyun.com/zh/quick-bi/user-guide/role-management)
- `组织用户管理`：管理员可以查询、筛选、批量更新、批量导出、编辑、删除、禁止/激活用户状态；还能看到账号配额和 Agent 配额使用情况。[管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)
- `工作空间管理`：Quick BI 用工作空间做资源隔离和协作边界。不同工作空间的资源互相隔离，组织管理员可以给每个工作空间配置独立管理权限。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
- `工作空间模式`：支持 `协同模式` 和 `私密模式`，本质上控制作品在作品列表中的默认可见范围。协同模式更适合共享协作，私密模式更适合按权限精细可见。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace) [选择工作空间模式](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspaces)
- `工作空间成员管理`：用户加入组织后，还要被加入工作空间才能创建作品；管理员可以添加、修改、查询和删除空间成员，并为其赋予空间角色。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace) [管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members)
- `用户组管理`：支持建立按部门、岗位、项目等维度组织的用户组，用于后续批量授权和批量维护；同一成员可以属于多个用户组，用户组最多支持 5 层类目。[创建用户组](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-user-group)
- `办公软件组织同步`：对于钉钉、企业微信、飞书等三方办公账号，可以通过“从办公软件同步”把组织架构和成员同步进来，减少重复建账号和重复建组的成本。[添加钉钉账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-a-dingtalk-account-as-an-organization-member) [添加企业微信账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-an-wecom-account-as-an-organization-member) [添加飞书账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-a-lark-account-as-an-organization-member) [从办公软件同步](https://help.aliyun.com/zh/quick-bi/user-guide/data-synchronization)
- `RAM 自动读取`：专业版支持直接读取当前账号体系下的 RAM 通讯录，适合阿里云原生组织快速导入用户。[通过RAM用户自动同步添加用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-users-by-using-a-ram-user)
- `个人账号与设备管理`：单个用户还可以在个人中心管理设备窗口、超额退出规则、绑定手机/邮箱/第三方办公账号，属于账号管理的个人侧补充能力。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 典型使用场景

- 企业首次上线 Quick BI，需要把阿里云账号、RAM 账号或办公软件成员批量纳入组织。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 按岗位把用户分成 `开发者 / 分析师 / 访问者`，避免领导账号被赋予开发能力，或开发账号被过度收缩成只读用户。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 按部门、区域、项目组建立用户组，方便后续一键分配空间成员或资源权限。[创建用户组](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-user-group)
- 按业务域建立多个工作空间，例如“销售分析空间”“财务分析空间”“供应链空间”，实现资源隔离和分团队管理。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
- 员工离职、调岗、账号变更时，管理员在组织用户页批量更新、停用或删除账号，并完成资源交接。[管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)
- 用户个人侧需要绑定钉钉/飞书/企业微信、清理多端登录窗口、处理账号切换时，用账号设置与管理页面完成。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 适用角色

- `组织管理员`：是账号管理的核心操作者，负责添加用户、建工作空间、管成员、删成员、设默认空间、管理工作空间模式等。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
- `权限管理员`：会参与组织角色和部分组织级管理，但更偏权限分配，后续 `权限管理` 章节会深入展开。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [角色管理](https://help.aliyun.com/zh/quick-bi/user-guide/role-management)
- `工作空间管理员`：更关注空间成员、空间模式、空间内协作边界和空间级角色。[管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members) [选择工作空间模式](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspaces)
- `普通成员`：主要受益于账号绑定、登录管理、工作空间加入和组织内协作分工。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 关键使用路径

1. 先确定组织要接入的账号来源：阿里云主账号、RAM 账号、RAM 角色，还是钉钉/企业微信/飞书/Teams 等办公账号。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
2. 组织管理员通过 `手动添加 / 批量导入 / RAM 自动读取 / 从办公软件同步` 将用户加入 Quick BI 组织。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [添加阿里云主账号为组织用户](https://help.aliyun.com/zh/document_detail/702450.html) [添加阿里云RAM账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-ram-users-as-organization-members) [通过RAM用户自动同步添加用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-users-by-using-a-ram-user)
3. 添加时同步定义 `用户类型`、`组织角色`，必要时给默认空间角色和所属用户组。[添加阿里云主账号为组织用户](https://help.aliyun.com/zh/document_detail/702450.html) [添加阿里云RAM账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-ram-users-as-organization-members)
4. 再创建或整理工作空间，设置所有者、空间模式和空间功能开关；必要时把某个空间设为默认工作空间，让新成员自动进入。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
5. 把用户或用户组加入具体工作空间，并指定空间角色。[管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members)
6. 后续在组织用户管理页持续做查询、编辑、批量更新、停用、删除和导出。[管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)
7. 最后由用户在个人中心做设备管理、登录规则和账号绑定收尾。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 前置条件 / 权限要求 / 版本限制

- `新建和管理工作空间`：仅高级版和专业版支持，且需要组织管理员权限。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
- `手动添加 / 批量导入阿里云主账号和 RAM 账号`：仅组织管理员支持，且仅适用于高级版和专业版。[添加阿里云主账号为组织用户](https://help.aliyun.com/zh/document_detail/702450.html) [添加阿里云RAM账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-ram-users-as-organization-members)
- `RAM 自动读取`：仅专业版支持，而且需要组织管理员权限；如果是 RAM 用户本人执行，还需要阿里云侧授予 `ListUserBasicInfos` 接口权限。[通过RAM用户自动同步添加用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-users-by-using-a-ram-user)
- `钉钉 / 企业微信 / 飞书账号导入`：仅组织管理员支持，且仅支持添加开通 Quick BI 专业版的用户。[添加钉钉账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-a-dingtalk-account-as-an-organization-member) [添加企业微信账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-an-wecom-account-as-an-organization-member) [添加飞书账号为组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/add-a-lark-account-as-an-organization-member)
- `工作空间成员加入限制`：只有 `开发者` 和 `分析师` 可以被添加为工作空间成员，`访问者` 不支持加入工作空间。[管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members)
- `空间角色匹配限制`：分析师不能被设为空间管理员或空间开发者，只能设为空间分析师或空间查看者；开发者才支持担任空间管理员或空间开发者。[管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members)
- `用户组层级限制`：用户组最多支持 5 层类目。[创建用户组](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-user-group)
- `同一账号组织归属限制`：同一用户账号只能加入一个 Quick BI 组织，若要加入新组织，必须先从原组织退出或被删除。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- `个人设备登录限制`：同一账号最多允许 3 个窗口同时登录；移动端 IM 登录不占限额，但移动端浏览器登录占限额。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)
- `第三方账号绑定`：仅专业版支持绑定第三方账号；办公软件导入的三方账号不能再绑定到当前账号，也不支持解绑。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)

## 与上下游点位的关系

- 它的上游是 `系统管理-仪表板管理` 的组织协作需求。因为只有先有成员体系、工作空间和账号分层，报表的分享、协同和治理才真正落得住。
- 它是后续 `系统管理-权限管理` 和整个 `权限管理` 章节的前置基础。没有组织成员、用户类型、空间成员、用户组，后面的角色授权、资源授权、行列权限都无法落地。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 它与后续 `安全配置` 也有关联，但边界不同：这一点更偏“身份与成员运营”，`安全配置` 更偏“组织级安全开关、访问控制和风险治理”。

## 限制与边界

- `账号管理` 负责的是“人和组织结构”进入系统，不等于“权限模型本身”。这里会提到组织角色和空间角色，但不会在这一点里展开讲每个权限点如何授权，那是后续章节内容。
- `工作空间模式` 主要影响的是作品列表可见范围，不直接改变作品本身权限。官方还明确提醒：新空间必须选定模式，后续不建议频繁切换，否则可能引起作品归属变化或可见性频繁变化。[选择工作空间模式](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspaces) [新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace)
- `删除用户` 不是简单移除。空间负责人不能直接删；删掉组织用户后，其组织内权限和内容会丢失，相关资源会默认转交给工作空间负责人。[管理组织用户](https://help.aliyun.com/zh/quick-bi/user-guide/manage-organization-members)
- `访问者` 这个类型非常适合纯查看场景，但它天然不能进入工作空间做创作或协作，这是账号管理阶段就要做好的角色边界设计。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management) [管理工作空间成员](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspace-members)

## 售前 / 方案视角价值解读

- 从产品定位看，Quick BI 的账号管理不是简单的“给账号开权限”，而是建立企业级 BI 组织结构的底座：账号来源统一、成员类型清晰、空间隔离明确、用户组可批量化、组织与个人侧账号管理并存。
- 从方案设计看，Quick BI 很适合“总部统一治理 + 多部门分空间协作”的企业场景。工作空间、空间模式、用户组和组织角色的组合，能同时兼顾共享协作和按权限隔离。[新建并管理工作空间](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-workspace) [选择工作空间模式](https://help.aliyun.com/zh/quick-bi/user-guide/manage-workspaces)
- 对客户最有说服力的一点是：Quick BI 不要求企业完全重新造一套账号体系，它能复用阿里云账号、RAM、钉钉、企业微信、飞书等现有身份体系，显著降低 BI 平台接入成本。[成员管理概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-member-management)
- 如果客户强调“账号安全”和“多端使用规范”，个人侧的设备管理、超额退出规则和三方账号绑定能力也能构成加分项。[用户登录](https://help.aliyun.com/zh/quick-bi/user-guide/user-logon)
