# 第3点：可视化分析 - 仪表板制作

这一点只覆盖 `可视化分析-仪表板制作`，重点讲“怎么把一个仪表板搭起来”。我会把 `交互分析 / 图表分析 / 查询控件 / 其他控件` 先当作后续深化能力，只在这里点到为止，不展开细讲。
官方对这一节的定义很明确：仪表板是 Quick BI 最典型的可视化分析载体，采用磁贴式布局，既能做数据展示，也能做筛选、查询和交互分析。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)

## 点位定义
`仪表板制作` 在 Quick BI 里不是“做一张图”，而是把 `数据集 + 多图表 + 页面布局 + 页面配置 + 预览验证` 组装成一份可持续使用的分析作品。官方整体流程里把它拆成：新建仪表板、替换数据集、配置字段、配置工具栏、配置标题区、配置全局参数、做可视化图表、再进入查询控件/其他控件/图表分析/主题设计等后续步骤。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)

## 这个点解决什么业务问题

- 把多个图表组织成一页面向业务决策的分析成品，而不是零散单图。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 让分析师在同一编辑界面里完成选数据、摆图表、调布局、配页面，而不需要在多个模块来回切换。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- 支持一份报表同时适配 `PC 端` 与 `移动端` 阅读场景。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- 在正式发布前，通过预览链接、版本恢复、自动保存等机制降低制作风险。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)

## 核心功能能力

- `整体制作流程清晰`：官方把仪表板制作定义为一条完整流水线，而不是碎片化功能集合，这对售前和实施都很重要，因为它说明 Quick BI 的仪表板是“作品型资源”。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- `多入口新建`：可以从空间外资源入口、空间内资源列表、仪表板模块、数据集列表、数据集编辑页等 5 个入口新建。前 3 种入口默认不带数据集，后 2 种会直接带入当前数据集字段，减少从“数据到报表”的跳转成本。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- `三段式编辑界面`：官方把仪表板编辑分成 `数据集选择区、仪表板配置区、仪表板展示区`。这意味着 Quick BI 的编辑器本身就是围绕“选数据、配图表、看成品”设计的。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- `顶部菜单栏负责全局制作动作`：包括编辑标题、撤销重做、切换 PC/移动端、替换数据集、全局参数、页面设置、预览、更多编辑后操作等，说明仪表板不是纯拖拽画布，而是带完整编辑工作流的报表编辑器。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `工具栏负责页面装配`：工具栏里除了添加图表，还提供查询控件、故事线、Tab、富文本、图片、内嵌页面、联动筛选面板、复用其他组件资源、页面栅格设置等，说明仪表板支持从“图表页”进化成“完整分析页面”。不过其中查询控件和其他控件我放到后续点详细讲。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `布局能力完整`：既支持拖拽改大小和位置，也支持全局布局缩放，还能统一调卡片间距和卡片内边距。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- `页面栅格设置`：这是较新的精细化布局能力，作用于整个仪表板页面的基础网格系统，可以统一控制栅格列数、单位高度和间距，适合 KPI 卡片并排、密集小图拼板等场景。[页面栅格设置](https://help.aliyun.com/zh/quick-bi/user-guide/page-grid-settings)
- `主题与页面样式`：仪表板支持官方主题、自定义主题、浅深色模式、图表色系、页面字体、圆角、间距、语义色、页面布局、标题区、故事大纲、页尾等全局样式配置。也就是说，Quick BI 的仪表板不仅能做“分析”，还能做统一品牌化视觉输出。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme)
- `移动端布局单独维护`：PC 与移动端可以分别调整布局，而不是简单响应式缩放。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `预览与版本回退`：编辑态支持 PC/移动端预览、复制预览链接、自动保存、恢复历史版本，适合报表交付前做校验和协作评审。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `复用组件`：可以从当前空间或模板市场复用已发布作品中的图表/控件组件，提高搭建效率。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)

## 典型使用场景

- 搭建日常经营分析看板、运营分析看板、专题分析页。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 把一批成熟图表和说明性控件拼装成一个对业务可直接阅读的分析页面。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- 为同一份报表分别设计 PC 与移动端展示方式，兼顾桌面分析和移动查看。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- 需要统一企业视觉风格、品牌配色和页面规范时，用主题设计和主题管理做标准化输出。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [报表主题管理](https://help.aliyun.com/zh/quick-bi/user-guide/manage-enterprise-recommended-themes)
- 需要快速复制成熟页面结构而不是从零开始时，用组件复用或模板市场加速搭建。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart) [查看并应用模板](https://help.aliyun.com/zh/quick-bi/user-guide/template-market)

## 适用角色

- `数据分析师 / BI 开发者`：是仪表板制作的主力角色，负责把数据集转成结构化分析页面。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- `售前 / 方案 / 业务分析负责人`：更关注主题、页面布局、移动端适配、预览和复用效率，因为这些直接决定报表交付质量。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [页面栅格设置](https://help.aliyun.com/zh/quick-bi/user-guide/page-grid-settings)
- `组织管理员`：在需要统一主题规范时会介入报表主题管理。[报表主题管理](https://help.aliyun.com/zh/quick-bi/user-guide/manage-enterprise-recommended-themes)

## 关键使用路径

1. 先从 5 类入口之一新建仪表板；如果从数据集入口进入，字段会直接带入右侧数据面板。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
2. 在编辑器里按 `数据集选择区 -> 配置区 -> 展示区` 的方式完成基础搭建。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
3. 先加图表并完成基础布局，再用顶部菜单栏处理标题、数据集替换、页面设置、全局参数等全局动作。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2) [配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
4. 用主题设计和页面栅格设置统一页面视觉和布局精度。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [页面栅格设置](https://help.aliyun.com/zh/quick-bi/user-guide/page-grid-settings)
5. 通过预览、预览链接、自动保存、历史版本恢复做交付前校验。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)

## 前置条件 / 权限要求 / 版本限制

- 新建仪表板本身可以不先选数据集，但正式搭建图表时仍要选择并配置数据集。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- `全局参数`：个人版不支持；高级版和专业版仅支持群空间；且必须在仪表板保存后才能设置。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `自定义主题`：Quick BI 专业版支持；组织管理员可新增自定义主题，非组织管理员只能选择已有自定义主题，不能新增。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [报表主题管理](https://help.aliyun.com/zh/quick-bi/user-guide/manage-enterprise-recommended-themes)
- `复制预览链接`：需要先在报表配置中打开“允许复制预览链接”；公开预览链接还要求组织级和空间级都允许公开作品。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `允许下载`：不只是报表内开关，还依赖组织管理和工作空间管理里的导出控制同时开启。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- `仪表板中图表及控件总数`：官方 FAQ 给出的限制是不超过 200 个。[常见问题](https://help.aliyun.com/zh/quick-bi/faq-2)

## 与上下游点位的关系

- 它的上游是 `数据集操作`。没有可用数据集，仪表板只是空画布。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2)
- 它是 `交互分析 / 图表分析 / 查询控件 / 其他控件` 的承载页。也就是说，这一点解决“把页搭起来”，后面几点评价的是“把页做深”。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 它和后面的 `系统管理-仪表板管理` 也要区分开：这一点讲“制作态”，后面那个点讲“制作完成后的管理态”，比如分享、公开、转让、删除、发布渠道等。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)

## 限制与边界

- `仪表板制作` 不等于 `图表深度配置`。官方流程中已把图表分析、查询控件、其他控件拆成单独环节，所以这里不宜把所有交互和分析能力都堆进“制作”。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 页面栅格设置是全局性的，不能对页面不同区域分别设置不同栅格；它改变的是布局最小单位，不是直接指定每个卡片的最终大小。[页面栅格设置](https://help.aliyun.com/zh/quick-bi/user-guide/page-grid-settings)
- 组件复用有限制：只能复用已发布作品中的组件，且暂不支持直接跨仪表板选择复制，只能先在同一仪表板完成组件选择后再复制到别的仪表板。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- 自动保存和历史版本恢复虽然好用，但保留上限明确：最多保留 3 个月内最近的 5 个发布版本和 15 个保存版本。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- 开启实时查询、同时开始自动刷会增加数据源压力，官方明确提示要谨慎开启。[配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)

## 售前 / 方案视角价值解读

- 从售前话术上，Quick BI 的仪表板不是“把图表堆在一页里”，而是一个完整的分析作品编辑器，兼顾页面结构、品牌样式、移动适配、预览验证和版本管理。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation) [配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
- 从方案设计看，它很适合承担企业里的“标准经营看板”与“专题分析页”角色，因为既支持统一主题，又支持多组件装配，还支持后续联动、钻取、查询控件深化。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 如果客户关心的是“报表能不能做得像企业产品页面，而不是数据截图”，Quick BI 的页面设置、主题设计、栅格布局、组件复用就是关键卖点。[主题设计](https://help.aliyun.com/zh/quick-bi/user-guide/design-a-theme) [页面栅格设置](https://help.aliyun.com/zh/quick-bi/user-guide/page-grid-settings)
- 如果客户关注交付效率，`从数据集直接新建仪表板`、`组件复用`、`自动保存`、`历史恢复` 这些能力能明显减少报表搭建与返工成本。[新建仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/create-a-dashboard-2) [配置图表工具栏区](https://help.aliyun.com/zh/quick-bi/user-guide/configure-the-toolbar-in-a-chart)
