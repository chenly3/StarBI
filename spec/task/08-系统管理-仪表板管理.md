# 第8点：系统管理 - 仪表板管理

这一点只覆盖 `系统管理-仪表板管理`。
它和前面的 `仪表板制作` 是两件事：前者解决“报表怎么搭出来”，这一点解决“报表做完之后怎么被保存、发布、共享、公开、嵌入、授权、回收和治理”。官方“管理仪表板”页面把这一点明确归纳为：`查看、查询、评论、移动、分享、发布渠道、公开、复制、转让和重命名、协同授权、收藏、删除` 等管理动作。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards) [发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)

## 点位定义
`仪表板管理` 本质上是仪表板的“运营治理层”。
它不再关注图表和控件怎么配置，而是关注一份仪表板从 `草稿 -> 已发布 -> 被分享/公开/嵌入 -> 被协同维护 -> 被下线/删除` 的完整生命周期管理。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)

## 这个点解决什么业务问题

- 让报表在“做好之后”真正进入组织使用，而不是停留在作者个人编辑页里。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- 让同一份报表可以按不同方式分发：私密分享、公开链接、移动端、OA 应用、嵌入第三方系统。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 让报表有可治理的资源属性，如目录归档、转让 Owner、重命名、删除、收藏、评论和协同授权。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 让报表变更有上线机制，而不是编辑完自动对所有用户生效。Quick BI 明确把保存和发布拆开，支持单独保存、发布、下线、重新发布。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)

## 核心功能能力

- `保存与发布分离`：Quick BI 明确把“保存”和“发布”拆开。仅保存时，变更对用户不可见；新报表需要“保存并发布”才上线；已发布报表修改后需要“重新发布”才同步给用户。这是典型的内容发布治理机制，而不是即时覆盖式编辑。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- `下线机制`：已发布仪表板可以被下线，下线后对用户不可见；删除已发布仪表板前，也必须先下线。这说明 Quick BI 区分“暂时撤回”和“永久删除”。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `管理页资源治理`：管理页支持搜索、筛选、排序、仅看我创建的报表、移动到目录、预览等，是典型的资源列表治理界面。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `分享能力`：支持私密链接分享给已有权限用户，也支持公开链接分享给任意访问者；还支持通过钉钉、飞书、企业微信等已绑定应用分发，并支持二维码在移动端查看和分享。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
- `公开能力`：公开后系统自动生成 URL，且无需登录阿里云账号即可访问；还支持设置截止日期或长期有效，并支持随时取消公开。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
- `发布渠道 / 嵌入渠道`：已发布仪表板可以开通嵌入，并配置发布渠道用于外嵌第三方系统。这说明 Quick BI 不只是“平台内看报表”，还支持把报表输出到外部业务系统使用。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards) [规格明细](https://help.aliyun.com/zh/quick-bi/support/specification-details)
- `协同授权`：报表支持多用户协作更新，但同一时刻只能有一个用户保存。官方提供抢锁机制，处理“多人同时编辑”时的冲突与覆盖问题。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `评论协作`：支持报表级评论和组件级评论，且支持 `@用户、表情、图片、回复、删除、评论筛选`，说明 Quick BI 的仪表板已经内置了轻量协作沟通能力。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `资源属性维护`：支持复制、移动、重命名、转让所有者、描述、收藏、深浅主题切换等，这些能力让仪表板更像正式可运营资源，而不是一次性文件。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `个人消费管理延伸`：除收藏整张仪表板外，Quick BI 还支持把不同仪表板中的单个数据卡片收集到个人 `卡片看板` 中，形成跨仪表板的个人关注页。这是面向阅读者侧的补充管理能力。[卡片看板](https://help.aliyun.com/zh/quick-bi/user-guide/pinboard)

## 典型使用场景

- 报表先在编辑态保存，再经过审批或确认后正式发布给业务用户查看。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- 正式上线后的报表通过私密链接分享给内部特定人员，通过公开链接分享给外部访客或无需登录的查看者。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
- 需要嵌入门户、营销系统、业务平台时，通过发布渠道开通嵌入。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards) [规格明细](https://help.aliyun.com/zh/quick-bi/support/specification-details)
- 多名分析师共同维护一份核心经营报表时，通过协同授权和抢锁机制控制冲突。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 业务方在预览态对图表提评论、@负责人、留痕讨论，减少线下沟通成本。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 高层或业务人员把散落在多份仪表板里的关键卡片收藏到个人卡片看板，形成“个人关注面板”。[卡片看板](https://help.aliyun.com/zh/quick-bi/user-guide/pinboard)

## 适用角色

- `分析师 / BI 开发者`：关注保存、发布、重新发布、下线、复制、协同编辑和分享。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard) [分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
- `空间管理员 / 资源 Owner`：关注转让、公开、协同授权、删除、目录移动和嵌入渠道治理。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `业务阅读者 / 管理层`：关注预览、评论、收藏、卡片看板和多渠道查看。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards) [卡片看板](https://help.aliyun.com/zh/quick-bi/user-guide/pinboard)
- `组织管理员 / 权限管理员`：关注组织级是否开放作品可公开、可授权、分享入口等开关，这是报表管理动作是否能落地的前提。[授权设置](https://help.aliyun.com/zh/quick-bi/user-guide/configure-collaborative-authorization)

## 关键使用路径

1. 报表制作完成后先执行 `保存`；如果要给别人看，再执行 `保存并发布`。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
2. 发布后，在仪表板管理页进行 `查询 / 预览 / 移动 / 重命名 / 转让 / 删除` 等基础资源管理。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
3. 需要分发时，根据对象选择：
   - 内部指定人员：私密分享。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
   - 外部访问或免登录访问：公开链接。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
   - OA/IM 协同：钉钉、飞书、企业微信分享。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
   - 第三方系统集成：发布渠道 / 嵌入配置。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
4. 需要多人维护时，开启协同授权；需要外部或更大范围传播时，先确认组织和空间都允许公开/授权。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards) [授权设置](https://help.aliyun.com/zh/quick-bi/user-guide/configure-collaborative-authorization)
5. 报表过时或需要撤回时，先 `下线`，确认不再使用后再 `删除`。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)

## 前置条件 / 权限要求 / 版本限制

- 管理页大部分动作都要求该仪表板已经创建并保存；分享和发布渠道则要求 `已发布` 状态。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `分享功能` 仅高级版和专业版支持，个人版不支持。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard)
- `转让` 仅适用于高级版和专业版。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 如果工作空间开启了 `作品发布审批`，保存并发布和重新发布都可能触发审批；空间管理员发布无需审批。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- 仪表板是否允许 `公开 / 分享 / 协同授权`，还取决于组织级和空间级功能权限。官方明确写了：如果组织级取消 `可公开 / 可授权`，所有工作空间里的资源都无法被授予公开、分享和协同授权权限。[授权设置](https://help.aliyun.com/zh/quick-bi/user-guide/configure-collaborative-authorization)
- 仪表板资源权限本身分为 `编辑 / 导出（含查看） / 查看` 三档，且权限存在包含关系：编辑包含导出和查看，导出包含查看。[仪表板权限](https://help.aliyun.com/zh/quick-bi/user-guide/dashboards-1)
- 发布渠道也有平台规格限制。官方规格页给出的 `报表外嵌发布渠道数` 限制值是 `10`。[规格明细](https://help.aliyun.com/zh/quick-bi/support/specification-details)

## 与上下游点位的关系

- 它的上游是 `可视化分析-仪表板制作`。上一点解决“如何把报表做出来”，这一点解决“做出来以后如何被组织管理和分发”。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 它和后面的 `系统管理-账号管理 / 权限管理 / 安全配置` 有明显边界：这里讲的是单个报表资源的管理动作；后面几个点会讲组织用户、角色、权限模型和安全开关这些更底层的管理规则。
- 它与后续 `权限管理` 章节有交叉但不重复。这里会提到“协同授权、查看/导出/编辑权限、公开/授权开关”这些前置条件，但不会在这一点里深入展开资源授权模型，后面会单独讲。

## 限制与边界

- `保存` 不等于 `上线`。Quick BI 明确要求通过发布或重新发布才能让变更对用户可见。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- `删除` 不是轻量动作。已发布报表要先下线；而且如果该报表被用于数据门户、订阅、监控告警，或被其他数据大屏、仪表板、电子表格做了跳转引用，删除时系统会提示受影响资源信息。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `协同授权` 虽然支持多人同时更新，但并不支持多人同时保存，最终仍通过抢锁机制收敛为单保存者模型。[管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- `公开与分享` 不是单看资源页面是否有按钮，还受组织和工作空间层面的治理开关约束。[授权设置](https://help.aliyun.com/zh/quick-bi/user-guide/configure-collaborative-authorization)
- `卡片看板` 是个人消费增强能力，不等于正式的仪表板发布渠道；它更适合个人视角汇总，不适合替代正式报表治理。[卡片看板](https://help.aliyun.com/zh/quick-bi/user-guide/pinboard)

## 售前 / 方案视角价值解读

- 从产品能力角度，Quick BI 的仪表板管理说明它不是“报表编辑器”而已，而是带有 `内容发布、分发、协同、运营治理` 的报表运营平台。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 从方案设计看，`保存/发布分离 + 审批 + 下线 + 重新发布` 这套机制很适合企业环境，因为企业报表往往需要版本控制、上线审核和回撤能力，而不是改完立刻覆盖全员查看结果。[发布仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/publish-a-dashboard)
- `分享 / 公开 / OA 分发 / 嵌入渠道` 说明 Quick BI 的报表可以覆盖平台内消费、移动消费、办公协同消费和第三方系统嵌入四类典型交付模式。[分享仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard) [管理仪表板](https://help.aliyun.com/zh/quick-bi/user-guide/manage-dashboards)
- 对企业客户而言，这一节最有说服力的点是：Quick BI 管的不是“图”，而是“报表资源生命周期”。这比单纯“能不能画图”更接近企业级 BI 落地需求。
