# 第14点：权限管理 - 集中授权

这一点只覆盖 `权限管理-集中授权`。
它和上一点 `资源权限` 的关系是：上一点讲“每类资源有哪些权限模型”，这一点讲“当资源很多、人很多时，Quick BI 怎么把这些权限高效地批量管起来”。官方对集中授权的定义很直接：除了单一资源授权外，还支持通过集中授权模块做多资源批量管理，并且支持 `从资源视角` 和 `从用户视角` 两种方式赋权。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)

## 点位定义
`集中授权` 本质上是 Quick BI 的“权限运营台”。
当企业资源数量和成员数量增大后，如果还靠逐个打开资源做协同授权，效率会很低，也很难做统一治理。集中授权就是把零散的资源权限管理提升到“批量运营”的层面。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)

## 这个点解决什么业务问题

- 多部门共用一个 BI 系统时，权限管理员可以从全局一次性看清资源和用户的权限状态，而不是到每个空间、每个资源里零散查找。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- 当要给一个团队批量开权限，或者把一个人的权限模板复制给另一个人时，能显著减少重复劳动。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- 当要统一治理某一类资源，比如某工作空间下所有仪表板、数据集、数据源时，可以从资源视角直接做全局控制，更安全也更高效。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
- 当组织要做“权限模板化”和“批量同步”时，集中授权是核心入口，因为它支持把一个资源或一个用户的权限配置同步到其他对象。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user) [从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)

## 核心功能能力

- `双视角集中授权`：Quick BI 把集中授权拆成两条主线：
  - `从资源赋权`：从资源到人管控权限。
  - `从用户赋权`：从用户到资源管控权限。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- `从资源赋权`：权限管理员能看到组织下所有资源，先选工作空间、再选资源类型与名称，然后统一设置编辑、查看、导出等权限及截止时间。从全局资源视角更适合做“资源安全治理”。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
- `从用户赋权`：从用户和用户组两个粒度展示某个主体在企业下所有工作空间的资源权限状态，还支持把权限一键同步到其他人。它更适合做“岗位模板复制”和“人员批量开通”。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- `支持对象范围广`：集中授权覆盖的数据对象不仅有 `数据门户、仪表板、数据大屏、电子表格、即席分析、自助取数、数据填报、数据集、数据源`，还覆盖 `经营洞察、指标洞察` 这类新型对象。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user) [从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
- `用户组视角支持`：按用户授权不仅能对单用户做集中赋权，还能对用户组统一开权限，这对按部门、岗位、项目组做批量治理很关键。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- `权限同步`：按用户授权支持“一键同步权限到其他用户”；按资源赋权也支持以当前资源权限为基准同步到同类资源。这使集中授权具备了“模板复制”能力，而不只是批量点击。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user) [同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)
- `同步模式区分`：从资源赋权的同步支持 `增量同步` 和 `覆盖同步`。
  - 增量同步：保留目标资源已有成员，只补充缺失权限。
  - 覆盖同步：目标资源被同步后的权限完全以当前资源为准，之前权限会被覆盖。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)
- `截止时间治理`：集中授权过程中可以直接设置查看或导出权限的截止时间，说明它不仅做批量发放，也兼顾权限时效治理。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)

## 典型使用场景

- 权限管理员要统一给销售部门一批成员开放某些仪表板、电子表格和数据门户的查看权限，适合走 `按用户授权`。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- 企业新来一位分析师，需要复制另一位分析师在各空间已有的资源权限，适合走 `按用户授权` 的权限同步。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- 要统一检查某工作空间下的所有仪表板、数据集和数据源是否开放过多，适合走 `从资源赋权`，从资源角度统一收口。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
- 一份权限配置成熟的标准报表，需要把相同权限同步到同类报表模板时，适合用 `同步资源权限`。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)
- 需要按用户组而不是逐个用户做授权时，集中授权是最自然的入口。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)

## 适用角色

- `权限管理员`：是集中授权的核心操作者，官方功能入口也明确写在权限管理员的控制台配置面板下。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- `组织管理员`：在全局治理或角色兼任情况下也会参与，但集中授权更偏权限管理员日常工作。
- `空间管理员 / 资源 Owner`：更多还是停留在单资源授权层面；一旦进入全局批量治理，就更适合权限管理员统一处理。

## 关键使用路径

1. 先判断当前任务更像“按人开权限”还是“按资源收权限”：
   - 如果目标是给某个人或某个用户组统一配齐资源权限，走 `按用户授权`。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
   - 如果目标是管某类资源对谁开放，走 `从资源赋权`。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
2. 在控制台 `集中授权` 模块进入对应入口。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
3. 选择视角下的对象：
   - 用户 / 用户组
   - 或 工作空间 / 资源类型 / 资源名称
4. 选择具体资源类型和权限档位，再设置是否有截止时间。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource)
5. 如果还要把当前权限模板复制出去，再执行同步：
   - `一键同步到其他用户`
   - 或 `同步到同类资源`
   - 并选择 `增量` 或 `覆盖` 模式。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user) [同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)

## 前置条件 / 权限要求 / 版本限制

- 官方明确写明：`权限管理员` 进入 Quick BI 控制台后，才能在配置面板下进入集中授权。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- 集中授权基于资源权限模型生效，所以前提仍然是组织级允许对应资源被授权，并且用户本身具备必要功能权限。[授权设置](https://help.aliyun.com/zh/quick-bi/user-guide/configure-collaborative-authorization) [角色管理](https://help.aliyun.com/zh/quick-bi/user-guide/role-management)
- 从用户赋权的“用户组”能力并不是对所有资源都一样。
  - 数据门户 / 数据作品 / 数据填报 / 经营洞察 / 指标洞察支持用户组赋权。
  - 数据集和数据源则主要按用户授权，不是按用户组统一发放。[按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- 从资源赋权的同步是“同类资源同步”，不是任意跨资源类型复制。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)
- 同步模式里 `覆盖同步` 风险明显更高，因为会直接覆盖目标资源原有权限；`增量同步` 更适合保守治理。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)

## 与上下游点位的关系

- 它的上游是 `资源权限`。如果不先理解各类资源能授什么权限，就无法正确使用集中授权做批量管理。
- 它和下一点 `数据行列权限` 是不同层次：
  - 集中授权解决“对象级批量赋权”。
  - 行列权限解决“数据细粒度可见范围”。
两者可以叠加，但不互相替代。
- 它和后面的 `申请与审批权限` 也有区别：
  - 集中授权偏主动治理和管理员批量发放。
  - 申请审批偏被动受理和权限申请闭环。

## 限制与边界

- `集中授权` 不会改变权限模型本身，它只是把已有权限模型批量化、运营化。
- 它也不是角色设计工具，不能替代 `角色权限`；角色仍负责功能边界，集中授权负责对象批量开放。[角色管理](https://help.aliyun.com/zh/quick-bi/user-guide/role-management)
- `按资源赋权` 更安全，但更像“资源管理员视角”；`按用户赋权` 更像“人力/部门模板视角”。两者没有绝对优劣，关键看你的治理目标。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- 同步能力虽然高效，但如果没有权限基线，很容易把不合理权限一起复制扩散。因此实施时应该先固化标准模板，再做同步。这一点是基于官方同步能力与覆盖模式作出的治理推论。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)

## 售前 / 方案视角价值解读

- 从售前角度，`集中授权` 是 Quick BI 企业级治理能力的重要证明。很多 BI 产品能做单资源分享，但不一定能把权限运营成“用户视角”和“资源视角”两套批量治理体系。[集中授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-centralized-authorization)
- 从方案设计看，这一能力尤其适合 `多部门、多空间、多资源、多用户` 的企业环境。资源量一上来，如果没有集中授权，权限管理成本会迅速失控。[从资源赋权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-resource) [按用户授权概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-granting-permissions-by-user)
- 对客户最有说服力的一点是：Quick BI 不只是“能授权限”，而是支持把权限做成“模板、批量、同步、时效”的运营机制，这更接近大型企业真正需要的权限治理方式。[同步资源权限](https://help.aliyun.com/zh/quick-bi/user-guide/sync-resource-permissions)
