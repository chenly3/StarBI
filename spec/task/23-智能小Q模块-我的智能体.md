# 第7点：智能小Q模块 - 我的智能体

本点覆盖页面：`我的智能体`目录页、`与智能体对话`、`智能体开发从0到1操作指南`，并结合 `智能体管理`、`知识库管理/企业知识库` 作为能力落地的关键前置说明。[我的智能体](https://help.aliyun.com/zh/quick-bi/user-guide/my-agents/)、[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)、[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)、[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)、[知识库管理](https://help.aliyun.com/zh/quick-bi/user-guide/knowledge-management/)、[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)

## 点位定义
`我的智能体` 是智能小Q里的“企业自定义 Agent 使用与交付入口”。它把企业自己构建的智能体纳入智能小Q统一会话框，支持直接调用、自动路由、持续对话和历史复用，不再只依赖官方内置 Agent。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)、[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)

## 这个点解决什么业务问题
- 解决“通用AI不懂企业术语和业务流程”的问题，让企业按场景构建专属 Agent。[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)
- 解决“智能分析能力无法按部门沉淀”的问题，通过智能体工坊+智能体形成组织化管理结构。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 解决“业务人员找不到该用哪个助手”的问题，超级框支持意图识别并路由到官方或自定义 Agent。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 解决“企业知识难以进入AI回答链路”的问题，可把知识文档和第三方知识库接入后在对话中调用。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)

## 核心功能能力
- 双入口调用：可从智能小Q超级框进入，也可从左侧 `自定义应用` 进入。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 导航固定：常用智能体可固定到左侧导航，降低高频使用路径成本。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 自动路由：超级框可按用户意图自动路由到官方 Agent 或自定义 Agent。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 多方式提问：支持示例问题、直接输入、快捷提问、语音输入，并支持中途停止回答。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 会话管理：支持查看历史对话与新建会话。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 工坊化管理：一个工坊可包含多个智能体；单工坊绑定一种开发平台，不同工坊资源隔离。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 多平台接入：工坊目前支持 Dify、阿里云百炼、阿里云百炼专属版模型。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 智能体发布控制：支持可用范围（全员或指定用户/用户组）、开场白与示例问题配置。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 生命周期管理：工坊可编辑/删除；智能体可编辑/停用/删除。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 知识增强：企业知识库支持本地文档、飞书文档、第三方知识库（百炼/通义点金/Dify），可配置召回参数 Top K 与 Score 阈值。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)

## 典型使用场景
- 销售拜访质检、风险客户识别等行业场景 Agent 化。[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)
- 各部门建立“部门专属助手”，业务人员直接提问拿结论。
- 企业把内部分析方法沉淀成可复用智能体，并在智能小Q统一调用。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 在问数/报告会话中开启企业知识库搜索，输出更贴合企业语境的结果。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)

## 适用角色
- 组织管理员：负责工坊、智能体、知识库接入与可用范围控制。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)、[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)
- 数据/产品/运营分析师：负责场景定义、提示词设计、示例问题设计和效果调优。[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)
- 业务用户：在“我的智能体”中消费已发布能力并持续对话。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)

## 关键使用路径
- 设计场景：先定义目标用户、输入数据、输出目标与评价标准。[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)
- 搭建工坊：在通用配置中创建智能体工坊并完成平台授权。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 新建智能体：配置类型、描述、示例问题、可用范围后发布。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 知识接入：按需接入企业知识文档或第三方知识库并设置召回参数。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)
- 用户消费：业务用户进入“我的智能体”提问、追问、看历史并复用高频会话。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)

## 前置条件 / 权限要求 / 版本限制
- `我的智能体` 属于增值能力，需先具备智能问数相关能力并分配权限。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 仅高级版、专业版支持；个人版不支持。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)、[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 使用端需具备智能体权限且具备小Q问数权限。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 管理端仅组织管理员可做智能体管理与企业知识库配置。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)、[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)
- `开发从0到1` 页面明确：调用 API 的开发调试能力仅专业版支持。[智能体开发从0到1操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/intelligence-development-from-0-to-1-operation-guide)

## 与上下游点位的关系
- 上游依赖：承接 `小Q搭建` 之后的“能力产品化”，依赖模型配置、权限体系、知识库治理。
- 横向协同：与 `小Q问数/小Q报告` 协同最强，知识库可在小Q对话框和小Q报告对话框被调用。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)
- 下游承接：下一点 `通用配置` 会更细讲大模型、AI功能管理、知识库与治理参数。

## 限制与边界
- 嵌入和无线端场景不支持“我的智能体”使用。[与智能体对话](https://help.aliyun.com/zh/quick-bi/user-guide/my-agent)
- 工坊层面是“单工坊单平台”，不同工坊资源隔离，不能直接互访。[智能体管理](https://help.aliyun.com/zh/quick-bi/user-guide/agent-management)
- 知识学习有边界：单次知识类型学习仅支持 3 万字符以内内容；本地文件类型和大小受限。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)
- 知识召回效果依赖 Top K/Score 阈值配置，需要运营调优，不是一次配置永久最优。[企业知识库](https://help.aliyun.com/zh/quick-bi/user-guide/enterprise-knowledge-base)

## 售前 / 方案视角价值解读
- 这个点的核心价值不是“再加一个聊天机器人”，而是“让企业把行业 Know-how 封装成可运营的智能应用”。
- 对客户决策层最有吸引力的是：能力可分工建设、可权限控制、可持续迭代，而不是个人临时 prompt。
- 对落地团队最关键的是两层抓手：`智能体工坊治理` + `企业知识库治理`，一个管应用结构，一个管内容质量。
- 对演示场景建议走闭环：`搭建工坊与智能体 -> 配知识 -> 在我的智能体中对话验证 -> 回到管理端持续优化`。
