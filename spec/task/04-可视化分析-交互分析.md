# 第4点：可视化分析 - 交互分析

这一点只覆盖 `可视化分析-交互分析`。
在 Quick BI 官方定义里，交互分析不是单一按钮功能，而是一组让“报表会响应用户动作”的能力集合。官方在交互式分析概述中把它明确拆成 `Story Builder（故事线）/ 钻取 / 联动 / 跳转 / 圈选 / 事件` 六类能力，用来支持用户动态与仪表板中的数据互动、快速定位问题并发现关联。[交互式分析概述](https://help.aliyun.com/zh/quick-bi/user-guide/interactive-analysis-overview)

## 点位定义
`交互分析` 的本质，是把静态可视化页面升级为“可探索、可导航、可下钻、可触发业务动作”的分析界面。
如果上一点 `仪表板制作` 解决的是“把页面搭出来”，那么这一点解决的是“让页面随着用户点击、选择、钻取和跳转产生反馈”。[交互式分析概述](https://help.aliyun.com/zh/quick-bi/user-guide/interactive-analysis-overview)

## 这个点解决什么业务问题

- 让管理层或业务人员从宏观页面逐步钻到细节，而不是只能看总览图。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
- 让一个图表中的选择自动影响其他图表，形成上下文一致的联动分析。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui)
- 让用户从当前图表直接跳到下一个报表、页面组件或外部系统，形成分析闭环。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
- 让分析不只停留在“看图”，还能通过圈选、事件、数据填报、钉钉触达把分析动作延伸到业务执行。[圈选](https://help.aliyun.com/zh/quick-bi/user-guide/multi-dimensional-data-visualization) [事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)
- 让单页报表具备叙事能力，使分析者按既定节奏引导读者理解结论，而不是自己在页面里摸索。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)

## 核心功能能力

- `故事线`：把同一张仪表板中的多个图表按逻辑顺序组织成一条分析路径，适合月度复盘、专题汇报、阶段性分析等“需要讲清楚分析思路”的场景。支持顶部横向和右侧纵向两种位置，支持自动勾选或手动勾选节点，节点最多 20 个，节点名称最长 50 个字符或汉字。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
- `钻取`：通过预先定义层级，把分析粒度从粗到细逐层展开。官方示例是区域 -> 省份 -> 城市。交叉表最多可设置 3 个钻取项，其他图表只能设置 1 个钻取项；最多支持 6 个钻取层级。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
- `联动`：一个图表被点击后，其他图表自动跟着变化。支持 `自动联动` 和 `手动联动` 两类配置；支持 `筛选联动` 与 `高亮联动` 两种被联动方式。自动联动主要用于同一数据集图表，手动联动适合定向配置。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui)
- `跳转`：支持三种跳转模式，分别是 `产品内跳转 / 页面内组件 / 外部链接`。其中产品内跳转可以跳到其他报表并做参数传递；页面内组件可以直接跳到当前页中的图表、Tab、富文本、图片、内嵌页面；外部链接可以拼接维度值跳去第三方系统。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
- `圈选`：一次选中多个数据点或单元格，然后执行只看、排除、查看数据、标注、钻取、联动、跳转等动作，本质上是“批量交互”。[圈选](https://help.aliyun.com/zh/quick-bi/user-guide/multi-dimensional-data-visualization)
- `事件`：目前只在交叉表中支持，包括 `数据填报事件` 和 `钉钉事件`。前者把数据写回目标数据库或表单，后者把数据通过 Ding、待办、日程发给钉钉用户，已经从“分析交互”延伸到“业务动作”。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)

## 典型使用场景

- 经营分析会中，从全国销售总盘下钻到区域、省份、城市定位问题，适合用 `钻取`。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
- 在地图、饼图或柱图里点选某个区域、品类、客户群后，让其余图表同步切换，适合用 `联动`。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui)
- 从总览报表跳到订单明细、专题页、外部业务系统，适合用 `跳转`。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
- 在复盘型报表里按“目标 vs 实际 -> 差异 -> 原因 -> 建议”的顺序讲故事，适合用 `故事线`。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
- 要一次性选中多类异常点再做排除、查看明细或批量跳转时，适合用 `圈选`。[圈选](https://help.aliyun.com/zh/quick-bi/user-guide/multi-dimensional-data-visualization)
- 需要在分析页面上直接触发“填报”或“联系负责人”的闭环动作时，适合用 `事件`。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)

## 适用角色

- `分析师 / BI 开发者`：是交互能力的主要配置者，负责把页面变成可探索界面。[交互式分析概述](https://help.aliyun.com/zh/quick-bi/user-guide/interactive-analysis-overview)
- `业务人员 / 管理层`：是交互能力的主要使用者，他们不一定会编辑报表，但会通过点击、筛选、钻取、跳转获取答案。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui) [钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
- `运营协同 / 业务执行人员`：在数据填报事件和钉钉事件场景下，交互能力也服务于执行侧协同。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)

## 关键使用路径

1. 先有已经制作完成的仪表板和图表，交互能力建立在已有页面基础上。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
2. 对于 `钻取`，先在数据集里准备好层次结构或可下钻字段，再在图表的数据区域配置钻取顺序。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
3. 对于 `联动`，从仪表板编辑页顶部的交互控制入口做全局联动设置，选择自动或手动，并定义发起联动与被联动的方式。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui)
4. 对于 `跳转`，在图表分析区绑定字段，选择跳转目标和打开方式，再配置全局参数映射、筛选条件和其他条件。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
5. 对于 `故事线`，先添加组件，再配置位置、样式、节点模式和节点文案。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
6. 对于 `事件`，先有交叉表，再在分析页签高级设置中新增事件，绑定填报表单或钉钉相关配置。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)

## 前置条件 / 权限要求 / 版本限制

- `故事线` 的前提是已创建仪表板。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
- `钻取` 依赖数据集层级准备；如果是地理钻取，相关字段要先转成地理信息并放进同一层次结构。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1)
- `产品内跳转` 如果目标报表没配置全局参数，就不能做关联跳转；并且 FAQ 说明全局参数仅群空间支持，且仪表板要先保存后才能设置。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink) [仪表板分析与交互FAQ](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard-1)
- `页面内组件跳转` 只是在当前页面内跳到图表或控件，不是跨报表导航。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
- `事件` 只支持交叉表；数据填报事件要求先创建数据填报，钉钉事件要求先开通钉钉微应用。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)
- `圈选` 不是所有图表都支持。趋势类、比较类、散点/气泡、词云/矩形树图支持较完整；交叉表和明细表仅支持只看、排除、复制。[圈选](https://help.aliyun.com/zh/quick-bi/user-guide/multi-dimensional-data-visualization)

## 与上下游点位的关系

- 它的上游是 `仪表板制作`。没有页面和图表，就没有交互对象。[仪表板制作概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-dashboard-creation)
- 它和后面的 `图表分析` 有配置入口上的重叠。因为很多交互能力最终会出现在图表分析的配置面板里，但从官方能力分类上看，`钻取 / 联动 / 跳转 / 圈选 / 事件 / 故事线` 明确属于 `交互分析` 的核心内容。[交互式分析概述](https://help.aliyun.com/zh/quick-bi/user-guide/interactive-analysis-overview)
- 它的下游价值，是把单张报表变成“问题定位和动作触发的界面”，为后续更精细的图表分析和查询控件配合打基础。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui) [跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)

## 限制与边界

- `钻取` 不支持批量下钻；通过第三方嵌入也不支持动态控制显示哪些钻取列，哪些维度可以钻取是固定的。[仪表板分析与交互FAQ](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard-1)
- `联动` 不是无限制自由组合。FAQ 明确写了：行列混合布局，或者行列都在一起时，不支持联动；没有维度的指标看板默认也不能设置联动，除非在过滤器里放一个维度；不支持 3 级联动。[仪表板分析与交互FAQ](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard-1)
- `高亮联动` 只支持部分图表，不是所有图表都能高亮联动。[联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui)
- `跳转` 虽然很强，但跨报表参数传递依赖目标报表先设计好全局参数；查询条件也不支持直接接收外部参数。[跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink) [仪表板分析与交互FAQ](https://help.aliyun.com/zh/quick-bi/user-guide/share-a-dashboard-1)
- `故事线` 也有结构限制。如果同一行有多个图表，只有最左侧图表能显示在故事节点列表里。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
- `事件` 的能力边界很清楚，它不是通用图表动作系统，目前只服务于交叉表。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)

## 售前 / 方案视角价值解读

- 从产品定位上看，Quick BI 的交互分析说明它不是“静态报表平台”，而是“可探索分析界面平台”。用户不是被动看图，而是能沿着数据上下文不断追问。[交互式分析概述](https://help.aliyun.com/zh/quick-bi/user-guide/interactive-analysis-overview)
- 从方案设计看，`钻取 + 联动 + 跳转` 是最典型的三件套：钻取解决“往下看”，联动解决“横向看”，跳转解决“去别处看”。这套组合足以覆盖绝大多数经营分析、专题分析、运营分析场景。[钻取](https://help.aliyun.com/zh/quick-bi/user-guide/drilling-filter-interaction-and-hyperlink-1) [联动](https://help.aliyun.com/zh/quick-bi/user-guide/filter-interaction-ui) [跳转](https://help.aliyun.com/zh/quick-bi/user-guide/hyperlink)
- `故事线` 是非常适合售前演示和管理汇报的能力，因为它把报表从“数据容器”变成“叙事容器”，能控制观看节奏和分析顺序。[故事线](https://help.aliyun.com/zh/quick-bi/user-guide/configure-a-story-builder-widget)
- `事件` 则体现了 Quick BI 不只是分析工具，还能把分析结果和业务动作衔接起来，尤其适合需要“看数后马上处理”的企业场景。[事件](https://help.aliyun.com/zh/quick-bi/user-guide/events)
