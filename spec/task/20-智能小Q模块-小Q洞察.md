# 第4点：智能小Q模块 - 小Q洞察

本点覆盖的官方页面主要有 4 类：`小Q洞察概述`、`指标洞察`、`经营洞察`、`小Q洞察查看`。这 4 类页面拼起来，才能看清小Q洞察不是单一功能，而是一套“先建洞察模型，再发布，再在智能小Q里消费”的完整机制。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)、[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)

## 点位定义
`小Q洞察` 是 Quick BI 在智能小Q体系里的“经营洞察应用层”。它不是临时问一句就结束的自然语言分析，也不是单纯对一个仪表板做文本解释，而是把 `指标洞察` 和 `经营洞察` 这两类可配置、可发布、可复用的洞察资产组织起来，最终在智能小Q里供业务人员查看、运行和分析。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)

## 这个点解决什么业务问题
- 解决“业务看到了异常，但不知道系统化怎么拆因”的问题。`指标洞察` 可以围绕单一指标做多维、多步归因分析，快速定位波动驱动因素。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 解决“企业经营分析缺少统一视角”的问题。`经营洞察` 把多个关键指标串成一个全局经营视图，不再只是看单张图表或单个指标。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 解决“分析结论一次性产出、无法持续运营”的问题。经营洞察支持发布、重新发布、协同授权和定期执行，说明它是可上线、可持续使用的洞察产品，而不是一次性分析草稿。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 解决“配置的人和消费的人不是同一批”的问题。官方把编辑、查询、查看权限分开，表明小Q洞察适合企业内部分工协作：分析人员负责设计，业务人员负责消费。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)

## 核心功能能力
- 双层洞察结构：官方明确小Q洞察由 `指标洞察` 和 `经营洞察` 两部分组成，前者偏指标归因，后者偏经营全景组织。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)
- 指标归因能力：指标洞察支持对单一数据集内指标变化做多维度、多步骤根因分析，并输出可视化归因分析报告。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 多种归因方法：支持维度归因和指标归因。维度归因包括 `单维分析`、`维度交叉`；指标归因包括 `指标相关性归因`、`指标因子归因`。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 灵活对比机制：洞察维度为日期时支持 `单日期对比` 和 `日期区间对比`，并能配置基准组、对照组；文本字段也支持基准组、对照组设置。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 指标资产复用：同一个指标洞察可以应用到不同经营洞察中，说明它本质上是一个可复用的分析组件。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 经营洞察编排能力：经营洞察支持在画布中新增 `指标节点` 和 `文本节点`，并按业务逻辑排版、连接，形成完整经营分析链路。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 结果内容定制：经营洞察中的指标节点可配置标题区、主要影响因素、归因摘要、详细分析；经营洞察整体也支持配置核心结论、分指标详情、行动建议，并支持自定义 Prompt。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 运行与调度能力：经营洞察支持即时运行，也支持 `定期洞察`，可按天、周、月执行，并可指定用户执行生成个性化结果。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 智能小Q消费入口：配置完成后，用户可以在智能小Q里直接查看经营洞察简报、执行过程、指标归因结果，并继续逐级展开分析节点。[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)

## 典型使用场景
- 经营复盘：把销售、利润、区域、渠道等关键指标组织成经营洞察，形成管理层可直接查看的经营简报。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 异常拆因：当指标出现明显波动时，先通过指标洞察定位贡献因素，再把结果纳入经营洞察统一呈现。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)、[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)
- 周期性经营巡检：使用定期洞察按天、周、月自动运行，适合例会、周报、月报类固定经营分析场景。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 分析中台对业务前台输出：数据团队负责配置和发布，业务团队在智能小Q里直接查看，不需要进入复杂编辑页面。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)

## 适用角色
- 数据分析师：负责建设指标洞察和经营洞察，是主要配置者。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 业务负责人/经营分析人员：在智能小Q中消费已发布的洞察结果，查看经营简报和归因过程。[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)
- 空间管理员/平台管理员：负责权限和协同授权体系，确保不同角色能按编辑、查询、查看分层使用。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)

## 关键使用路径
- 第一步：先建 `指标洞察`。选择数据集和指标，配置聚合方式、洞察维度、对比方式、过滤条件，再设定维度归因和指标归因逻辑。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 第二步：测试并发布指标洞察。发布后，它才能作为经营洞察里的可选分析节点被引用。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 第三步：新建 `经营洞察`。在画布上拖入指标节点和文本节点，按业务逻辑编排，并配置简报展示内容。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 第四步：配置洞察参数与全局展示。包括基准组/对照组、查询条件、简报内容、自定义 Prompt 等。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 第五步：保存并发布，可按需开启 `定期洞察`。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 第六步：业务用户到 `智能小Q -> 小Q洞察` 中查看结果，必要时运行刷新、查看执行过程、展开指标节点继续追因。[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)

## 前置条件 / 权限要求 / 版本限制
- 小Q洞察属于增值模块，需要额外购买。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 使用前需要先配置好 `指标洞察` 和 `经营洞察`，否则智能小Q中的小Q洞察入口只能进入查看框架，无法实际消费结果。[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)
- 指标洞察要求用户拥有 `数据集使用权限` 和 `指标洞察新建（编辑）权限`。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 经营洞察要求用户拥有 `数据集使用权限` 和 `经营洞察新建（编辑）权限`。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 官方将经营洞察权限分成 `编辑 / 查询 / 查看` 三层：
- `编辑`：可编辑、即时洞察、查看简报。
- `查询`：可即时洞察、查看简报。
- `查看`：仅可查看已生成简报。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)
- 指标洞察权限分成 `编辑 / 查看` 两层。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)

## 与上下游点位的关系
- 上游依赖：它依赖前面的 `智能小Q操作指南` 作为统一入口，也依赖数据集、指标设计和权限体系作为基础。[Quick BI 智能小Q操作指南](https://help.aliyun.com/zh/quick-bi/user-guide/smart-q-home-page)、[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 与 `小Q问数` 的关系：小Q问数偏即时自然语言取数，小Q洞察偏预先配置好的经营分析资产消费，两者一个偏“临场分析”，一个偏“产品化洞察”。这是基于官方页面结构做出的产品定位判断。[发起问数](https://help.aliyun.com/zh/quick-bi/user-guide/user-guide-for-smart-q-a)、[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)
- 与 `小Q解读` 的关系：小Q解读更像对现有仪表板做自动解释；小Q洞察更像事先建好的经营分析应用，强调指标编排、发布、协作和周期性执行。[小Q解读概述](https://help.aliyun.com/zh/quick-bi/user-guide/overview-of-small-q-interpretation)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 下游承接：小Q洞察把“数据分析能力”进一步包装成“经营洞察产品”，更适合长期给业务和管理层稳定消费。

## 限制与边界
- 指标洞察的分析对象是 `单一数据集内的单一指标`，不是跨多个数据集的自由组合分析。[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 经营洞察不是完全自由生成，它依赖前面已经配置和发布好的指标洞察；从官方页面看，经营洞察中的指标节点只能选择已发布的指标洞察。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 查看侧如果当前没有最新结果，需要手动 `运行` 才能刷新，不是所有场景都实时自动更新。[小Q洞察查看](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-operation-instructions)
- 指定用户执行定期洞察后，结果是个性化的，并且仅指定用户可查看，这在组织共享场景下要提前设计好权限策略。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)
- 从官方描述判断，小Q洞察更适合“已标准化、需持续消费”的经营分析场景，不适合替代全部探索式分析。这一条是根据其配置、发布、授权、定期执行机制做出的产品判断。[小Q洞察概述](https://help.aliyun.com/zh/quick-bi/user-guide/small-q-insight-overview)、[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)

## 售前 / 方案视角价值解读
- 小Q洞察最值得讲的不是“AI会分析”，而是“AI分析结果可以被产品化运营”。它有配置、发布、查看、协同、下线、定期执行这些完整管理动作，明显比一次性 AI 问答更适合企业落地。[经营洞察](https://help.aliyun.com/zh/quick-bi/user-guide/business-insight)、[指标洞察](https://help.aliyun.com/zh/quick-bi/user-guide/attribution-analysis)
- 对客户高层来说，这个点可以被理解成“经营驾驶舱上的智能洞察层”，把关键指标变化、原因和结论结构化输出出来。
- 对分析团队来说，小Q洞察相当于把专家经验沉淀成可发布的洞察资产，减少反复做同类经营分析的成本。
- 对售前演示来说，最好的讲法是三段式：`先做指标归因 -> 再编排经营洞察 -> 最后在智能小Q里查看和运行`，这样能把“建设侧”和“消费侧”都讲完整。
- 如果客户已经有成熟 BI 报表体系，小Q洞察的价值尤其明显，因为它补的是“从报表到经营分析产品”的那一层，而不是推翻原有报表体系重来。
