# 问数配置 Calicat 原型视觉还原设计

## 1. 背景

当前 `系统设置 -> 问数配置` 已具备基础功能链路，但页面视觉与 Calicat 原型差距较大，尤其体现在：

- `分析主题管理` 主页面的左右布局、表格密度、搜索框/按钮尺寸、主题列表选中态与原型不一致。
- `新建分析主题`、`添加问数资源`、`数据预览` 三个弹窗的标题层级、留白、表头、按钮位置与原型不一致。
- 用户当前明确要求：使用 `agent-browser` 打开 Calicat 原型，对这 4 个界面做全面视觉优化。

## 2. 目标

本次仅围绕以下 4 个 Calicat 原型做视觉 1:1 优先还原，真实数据链路保持不变：

## 3. 明确范围

### 3.1 在范围内

- `问数配置` 页中的 `分析主题管理` 视图视觉还原
- `新建分析主题` 弹窗视觉还原
- `添加问数资源` 弹窗视觉还原
- `数据预览` 弹窗视觉还原
- 使用 `agent-browser` 打开原型与当前页面，逐轮比对后收紧样式

### 3.2 不在范围内

- 不扩展顶部其它导航页：`小Q问数`、`智能体管理`、`仪表板管理`、`权限管理`、`企业知识库管理`、`嵌入管理`、`问数运营`、`全局配置`
- 不改 `用户管理`、`组织管理`、其它系统设置菜单
- 不新增业务功能、权限模型、知识库体系
- 不伪造示例数据，不为贴图重写数据源
- 不把真实页面改造成静态原型页

## 4. 还原原则

### 4.1 视觉优先，数据真实

- 视觉骨架、间距、字体层级、按钮尺寸、表头样式、弹窗尺寸尽量对齐原型。
- 列表和弹窗中的数据继续使用现有接口返回的真实数据。
- 原型中的示例数据只作为布局参考，不要求逐项模拟。

### 4.2 局部微调，不重搭整页

- 优先复用现有 `query-config/index.vue` 和 `query-theme/index.vue` 结构。
- 仅在现有结构无法承载原型时，做有限的模板层调整。
- 不重写整页信息架构，不额外拆出无必要的新页面。

### 4.3 不伤功能链路

- 主题创建、资源分配、数据预览继续走当前逻辑。
- 视觉优化不能改变现有接口入参与数据保存行为。
- 如果视觉还原与现有业务行为冲突，以不破坏现有业务链路为优先。

## 5. 目标界面拆解

### 5.1 分析主题管理主页面

目标还原点：

- 左侧卡片式 `分析主题列表` 容器
- 左侧顶部标题与蓝色圆形 `+` 按钮
- 左侧搜索框的高度、圆角、图标位置
- 主题列表项的常态、选中态、更多操作按钮样式
- 右侧 `问数资源` 容器标题、副标题、搜索框、蓝色主按钮
- 表头灰底、列宽、行高、复选框、状态图标、行内图标和删除按钮
- 整体白底卡片、边框、阴影、内容留白

不要求：

- 左右数据项与原型示例完全相同
- 顶部其它管理页导航一起实现

### 5.2 新建分析主题弹窗

目标还原点：

- 弹窗宽度、标题位置、关闭按钮位置
- 表单标签字号和必填星号样式
- 输入框圆角、高度、计数位置
- 文本域高度、边框、底部计数
- 底部 `取消 / 确定` 按钮尺寸、对齐和间距

### 5.3 添加问数资源弹窗

目标还原点：

- 标题和关闭按钮
- 顶部搜索框尺寸、图标、placeholder
- 资源列表行高、复选框、图标、名称排布
- 列表区域留白与滚动区域密度
- 底部按钮位置与视觉层级

### 5.4 数据预览弹窗

目标还原点：

- 顶部标题行与关闭按钮
- 蓝色维度信息条
- 表格头部样式、行高、边框、字体层级
- 表格容器宽度、高度、底部留白
- 底部关闭按钮位置

## 8. 风险与约束

- Calicat 原型示意中存在示例数据与真实环境数据不一致的问题，因此只能保证布局/视觉接近，不保证内容逐字逐项一致。
- `agent-browser` 和浏览器会话可能偶发卡顿，验证时应尽量串行执行，避免并发浏览器状态互相污染。
- 如果原型中某些元素与现有真实功能结构冲突，应保留真实功能结构，仅在视觉层做最大程度贴近。
