# 权限配置原型设计说明

## 1. 文档目标

本文档用于指导 `StarBI / DataEase 二开项目` 的 `权限配置` 模块原型设计。文档目标不是定义技术实现细节，而是把 DataEase 官方权限配置能力整理成可直接用于 Figma 出稿、产品评审和研发理解的页面与交互说明。

> 2026-04-23 更新口径：
> `权限配置` 模块当前只承载 `菜单权限 / 资源权限`；
> `行权限 / 列权限` 迁移为 `系统设置 -> 问数配置 -> 问数资源管理` 下的数据集级页面。

本文档要同时解决三件事：

- 给设计师一套可直接落原型的页面结构、区域组成、组件状态和交互流程
- 给产品一套明确的权限边界、角色差异、异常场景和保存策略
- 给研发一套清晰的页面入口、授权对象、资源对象和双视角映射说明

## 2. 设计基线与输入依据

### 2.1 官方能力基线

本模块严格对标 DataEase 官方权限配置能力，原型能力边界采用以下基线：

- 权限配置首页
- 按用户配置
- 按资源配置
- 菜单权限
- 资源权限
- 行权限
- 列权限
- 白名单配置
- 脱敏规则配置

官方能力参考：

- DataEase 功能介绍：`https://dataease.cn/features.html`
- DataEase 企业版介绍：`https://dataease.cn/pro.html`

### 2.2 仓库现有接口基线

- 菜单/资源权限接口：[AuthApi.java](/Users/chenliyong/AI/github/StarBI/dataease/sdk/api/api-permissions/src/main/java/io/dataease/api/permissions/auth/api/AuthApi.java)
- 行权限接口：[RowPermissionsApi.java](/Users/chenliyong/AI/github/StarBI/dataease/sdk/api/api-permissions/src/main/java/io/dataease/api/permissions/dataset/api/RowPermissionsApi.java)
- 列权限接口：[ColumnPermissionsApi.java](/Users/chenliyong/AI/github/StarBI/dataease/sdk/api/api-permissions/src/main/java/io/dataease/api/permissions/dataset/api/ColumnPermissionsApi.java)

这些接口共同说明当前产品能力是：

- 存在 `按对象查询权限` 与 `按资源查询权限` 的双视角接口
- 菜单权限和资源权限已有单独保存接口
- 行权限和列权限已有独立数据集级接口
- 当前组织上下文已是用户与角色查询的基础契约

### 2.3 已确认的需求边界

本次原型说明已确认以下口径，设计中不再重复讨论：

- 严格按官方五块能力做一期基线，不裁剪
- 菜单权限只授予角色
- 资源权限可授予角色或用户
- 行权限支持角色、用户、系统变量和白名单
- 列权限支持禁用、系统内置脱敏、自定义脱敏和效果预览
- 所有授权、查询、保存都基于当前组织上下文
- 默认不能跨组织配置
- `按用户配置` 与 `按资源配置` 是同一份权限数据的双视角映射
- 任一视角保存后，另一视角查询结果必须一致
- 页面需覆盖系统管理员和组织管理员两类角色
- 权限配置主页面仅保留菜单权限、资源权限两类页签
- 行权限、列权限从权限配置模块中迁出，改为从问数资源管理的数据集操作入口进入
- 页面必须显式展示直接授权与继承授权的区别
- 各类权限页都要定义空状态、无权限状态、权限不足状态
- 所有修改采用显式保存，不做自动保存
- 有 Calicat 原型的页面，后续视觉实现以 Calicat 为基线；没有原型覆盖的部分，不自行发散

## 3. 适用角色与组织边界

### 3.1 适用角色

本模块原型至少覆盖以下两类角色：

- 系统管理员
- 组织管理员

### 3.2 系统管理员权限边界

系统管理员在原型中的能力定义为：

- 可进入权限配置首页
- 可在有权限的组织上下文内查看和配置菜单权限、资源权限
- 可在问数资源管理中按数据集进入行权限、列权限页面
- 可查看角色、用户、资源、数据集、字段等完整对象集合
- 可执行新增授权、移除授权、修改规则、维护白名单、维护脱敏规则

### 3.3 组织管理员权限边界

组织管理员在原型中的能力定义为：

- 可进入权限配置首页
- 页面结构与系统管理员基本一致
- 仅可查看和管理当前组织权限范围内的角色、用户、资源和数据集
- 不可越过当前组织对其他组织对象进行配置
- 若超出权限范围，不直接隐藏到不可理解，而是显示只读或无权限提示

### 3.4 当前组织上下文原则

权限配置必须持续依附于“当前组织”。

设计要求：

- 页面头部或主区域固定展示当前组织名称
- 如系统支持组织切换，切换后需整体刷新权限配置上下文
- 所有查询、树加载、对象列表、保存动作仅作用于当前组织
- 不允许在同一次操作中同时配置多个组织

## 4. 产品定位与页面目标

权限配置是组织管理中心中负责定义“谁可以访问什么、以什么方式访问、数据看见什么范围、字段显示成什么样”的控制模块。

它与用户管理、组织管理的关系如下：

- 组织管理定义组织边界
- 用户管理定义用户与角色
- 权限配置定义角色、用户、资源和数据集的实际访问结果

本模块在原型中要让用户明确理解四类能力：

- 菜单权限：是否能进入某个菜单或功能入口
- 资源权限：是否能查看、管理、授权、导出某类资源
- 行权限：能看到数据集中的哪些数据行
- 列权限：字段是否可见，以及是否需要脱敏

## 6. 全局页面设计原则

### 6.1 先定上下文，再改权限

所有页面必须遵循：

- 先识别当前组织
- 再识别当前视角
- 再识别当前授权对象或资源对象
- 最后进入权限编辑

### 6.2 明确区分来源

页面必须显式表达：

- 直接授权
- 继承授权
- 当前用户不可编辑的继承结果

用户不仅要看到最终生效结果，还要知道来源。

### 6.3 显式保存

所有页面统一采用显式保存：

- 勾选、取消勾选、修改规则后，仅进入待保存状态
- 页面必须有明确的 `保存` 按钮
- 离开页面、切换对象、切换视角、切换页签时，如存在未保存修改，需要弹出确认提示
- 保存成功后需有明确反馈
- 保存失败时需保留用户当前输入

### 6.4 双视角一致

`按用户配置` 和 `按资源配置` 看到的是同一份权限数据：

- 任一视角保存后，另一视角查询应体现同样结果
- 文案上要清楚表达“我现在是在看对象”还是“我现在是在看资源”

### 6.5 必备状态

每类权限页必须覆盖：

- 正常态
- 未选对象/未选资源态
- 空状态
- 无权限状态
- 权限不足只读态
- 加载态
- 保存成功态
- 保存失败态
- 未保存离开提示态

## 7. 权限配置首页

### 7.1 页面目标

首页不是编辑页，而是入口页。它用于帮助用户理解模块边界，并选择从哪个视角进入配置。

### 7.2 页面组成

- 页面标题区
- 当前组织上下文区
- 双视角入口区
- 权限能力说明区
- 风险提示区

### 7.3 页面元素说明

**页面标题区**

- 标题：`权限配置`
- 副标题：可选，用于说明“配置当前组织下对象与资源的访问权限”

**当前组织区**

- 显示当前组织名称
- 如支持组织切换，可提供切换入口

**双视角入口区**

- 入口一：`按用户配置`
- 入口二：`按资源配置`

每个入口卡片建议包含：

- 名称
- 一句话说明
- 典型使用场景
- 点击进入按钮

**权限能力说明区**

- 菜单权限：控制角色是否可访问功能菜单
- 资源权限：控制角色或用户是否可操作业务资源
- 行权限：控制数据集可见数据范围
- 列权限：控制字段可见与脱敏规则

**风险提示区**

- 所有配置基于当前组织生效
- 两个视角展示的是同一份权限数据
- 修改后需手动保存

### 7.4 首页交互

- 点击 `按用户配置` 进入按用户配置视图
- 点击 `按资源配置` 进入按资源配置视图
- 切换组织后首页信息刷新
- 无权限用户进入首页时展示无权限状态

## 8. 按用户配置视图

### 8.1 页面目标

从“角色/用户”出发查看并编辑其拥有的权限，回答“这个对象现在有什么权限”和“我要改它什么权限”。

### 8.2 页面组成

- 页面标题与当前组织区
- 视角切换区
- 对象类型切换区
- 对象搜索筛选区
- 授权对象列表区
- 权限分类页签区
- 权限编辑区
- 底部保存区

### 8.3 对象类型规则

- 菜单权限：对象类型仅支持角色
- 资源权限：对象类型支持角色、用户
- 行权限：对象类型支持角色、用户、系统变量
- 列权限：对象类型按角色或用户进行配置

### 8.4 列表与筛选建议

左侧对象列表建议支持：

- 对象类型切换：角色 / 用户
- 搜索
- 当前组织范围筛选
- 自定义角色筛选
- 仅显示已配置权限对象

列表字段建议包含：

- 对象名称
- 对象类型
- 继承来源或角色类型
- 权限状态标记

### 8.5 主交互流程

1. 进入 `按用户配置`
2. 选择对象类型
3. 搜索并选中角色/用户
4. 右侧显示当前对象的权限页签内容
5. 进行勾选、取消勾选或规则编辑
6. 页面进入待保存状态
7. 点击保存
8. 保存成功后提示成功并更新数据

### 8.6 页面状态

- 未选对象：右侧提示“请先选择角色或用户”
- 空列表：提示当前组织暂无可配置对象
- 无权限：对象可见但不可编辑，右侧进入只读态
- 未保存离开：切换对象或视角前提示确认

## 9. 按资源配置视图

### 9.1 页面目标

从“菜单、资源、数据集、字段”出发查看授权对象，回答“这个资源当前授权给了谁”和“我要给这个资源增加或移除哪些对象”。

### 9.2 页面组成

- 页面标题与当前组织区
- 视角切换区
- 资源搜索筛选区
- 资源树/资源列表区
- 已授权对象区
- 权限分类页签区
- 底部保存区

### 9.3 资源对象规则

- 菜单权限：以菜单树节点作为资源对象
- 资源权限：以业务资源树或资源分组作为资源对象
- 行权限：以数据集作为资源对象
- 列权限：以数据集及字段层级作为资源对象

### 9.4 主交互流程

1. 进入 `按资源配置`
2. 在左侧资源树中选择资源
3. 右侧展示该资源已授权对象
4. 新增、移除或调整授权
5. 页面进入待保存状态
6. 点击保存
7. 保存后刷新授权对象结果

### 9.5 页面状态

- 未选资源：提示“请先选择资源”
- 空资源：提示当前组织暂无可配置资源
- 空授权：提示当前资源尚未授权给任何对象
- 只读态：用户可查看结果但无权修改

## 10. 统一权限配置主页面

### 10.1 页面定位

权限配置模块共用一套主页面壳，仅通过页签切换 `菜单权限 / 资源权限`。

### 10.2 固定区域

以下区域在两类权限页签中保持一致：

- 页面标题区
- 当前组织区
- 当前视角标识区
- 当前对象/当前资源标识区
- 页签切换区
- 保存操作区

### 10.3 页签切换规则

- 菜单权限
- 资源权限

页签切换时：

- 如无未保存修改，可直接切换
- 如有未保存修改，弹出确认提示
- 当前对象或资源上下文保留，但展示内容按目标页签刷新

## 11. 菜单权限配置

### 11.1 页面目标

配置角色是否可访问某一菜单或功能入口。

### 11.2 核心规则

- 菜单权限只支持角色，不支持用户
- 菜单结构采用树形展示
- 页面需表达继承状态
- 当前视角不同，仅对象选择方式不同，最终数据一致

### 11.3 页面结构建议

**按用户配置进入时**

- 左侧：角色列表
- 右侧：菜单树 + 勾选状态

**按资源配置进入时**

- 左侧：菜单树
- 右侧：已授权角色列表

### 11.4 交互要求

- 支持树形展开/收起
- 支持勾选授权
- 支持取消授权
- 支持显示部分选中状态
- 继承授权节点不可直接编辑时，要给出原因

### 11.5 状态要求

- 未选角色/未选菜单状态
- 空菜单树状态
- 无权限状态
- 待保存状态

## 12. 资源权限配置

### 12.1 页面目标

配置角色或用户对业务资源的访问权限。

### 12.2 核心规则

- 资源权限可授予角色或用户
- 资源按业务分组或资源树展示
- 权限维度至少区分：
  - 查看
  - 管理
  - 授权
  - 导出
- 页面必须显式展示继承与来源

### 12.3 页面结构建议

**按用户配置进入时**

- 左侧：对象列表
- 右侧：资源树 + 权限维度勾选矩阵

**按资源配置进入时**

- 左侧：资源树
- 右侧：已授权对象列表 + 各对象的权限维度

### 12.4 交互要求

- 支持对象新增与移除
- 支持单资源多维度勾选
- 支持直接授权与继承授权的区分展示
- 支持空授权资源从零开始配置

### 12.5 状态要求

- 未选对象/资源状态
- 当前资源无授权对象状态
- 当前对象无资源权限状态
- 只读态

## 13. 行权限配置

### 13.1 页面目标

定义单个数据集在查询时的数据行可见范围。

### 13.2 核心规则

- 行权限支持角色、用户、系统变量
- 行权限支持白名单
- 规则配置采用条件表达式或规则配置区
- 行权限以数据集为唯一页面上下文
- 页面入口位于 `系统设置 -> 问数配置 -> 问数资源管理` 的数据集操作菜单

### 13.3 页面结构建议

- 顶部：返回问数资源管理 + 当前数据集名称
- 主区：规则列表 / 空状态 / 弹窗入口
- 弹窗：角色、用户、系统变量三类规则配置 + 白名单配置

### 13.4 页面元素说明

- 规则名称，可选
- 授权对象
- 数据集名称
- 条件配置区
- 条件预览区
- 白名单入口
- 规则摘要

### 13.5 交互要求

- 支持新增行权限规则
- 支持编辑既有规则
- 支持删除规则
- 支持打开白名单配置弹窗
- 支持区分系统变量规则与普通对象规则

### 13.6 状态要求

- 当前数据集上下文缺失或不支持
- 当前无行权限规则
- 规则校验失败
- 白名单为空
- 保存成功/失败

## 14. 列权限配置

### 14.1 页面目标

定义单个数据集字段是否可见，以及显示时是否需要脱敏。

### 14.2 核心规则

- 列权限支持字段级配置
- 列权限至少支持两类策略：
  - 禁用
  - 脱敏
- 脱敏支持两类来源：
  - 系统内置规则
  - 自定义规则
- 页面需提供效果预览
- 页面入口位于 `系统设置 -> 问数配置 -> 问数资源管理` 的数据集操作菜单

### 14.3 页面结构建议

- 顶部：返回问数资源管理 + 当前数据集名称
- 主区：字段权限规则列表 / 空状态 / 弹窗入口
- 弹窗：角色或用户对象选择、字段策略配置、脱敏规则配置

### 14.4 页面元素说明

- 数据集选择区
- 字段列表
- 字段类型
- 当前策略状态
- 策略切换控件：正常 / 禁用 / 脱敏
- 脱敏规则入口
- 预览区

### 14.5 交互要求

- 支持批量查看字段
- 支持单字段策略切换
- 选择脱敏时可打开脱敏规则配置弹窗
- 切换规则后预览效果即时更新
- 保存前不立即生效

### 14.6 状态要求

- 未选数据集
- 数据集无字段
- 当前字段未设置策略
- 脱敏规则不完整
- 预览不可生成

## 15. 白名单配置弹窗

### 15.1 使用场景

### 15.2 页面目标

### 15.3 弹窗组成

- 弹窗标题
- 当前规则信息区
- 白名单用户搜索区
- 白名单已选区
- 保存/取消区

### 15.4 交互要求

- 支持按用户名搜索
- 支持添加和移除白名单成员
- 支持查看当前已选白名单
- 保存后回写到行权限页面

### 15.5 状态要求

- 空白名单状态
- 搜索无结果
- 无权限维护白名单
- 保存成功/失败

## 16. 脱敏规则配置弹窗

### 16.1 使用场景

### 16.2 页面目标

让用户为某字段选择或定义脱敏规则，并能看到预览效果。

### 16.3 弹窗组成

## 17. 全局反馈与异常场景

### 17.1 成功反馈

以下动作需有明确成功提示：

- 保存菜单权限
- 保存资源权限
- 保存行权限
- 保存列权限
- 保存白名单
- 保存脱敏规则

### 17.2 失败反馈

失败时至少区分：

- 权限不足
- 数据校验失败
- 当前对象已失效
- 当前资源已失效
- 服务异常

失败提示不能只给通用报错，需尽量让用户知道能否继续修正。

### 17.3 未保存离开提示

以下动作都需触发离开确认：

- 切换视角
- 切换对象
- 切换资源
- 切换页签
- 切换组织
- 关闭页面

建议提示文案统一：

- `当前修改尚未保存，是否确认离开？`

### 17.4 无权限提示

无权限状态需明确指出：

- 是没有进入当前模块的权限
- 还是没有编辑当前对象/资源的权限

页面层面建议区分：

- 模块级无权限：整个页面不可进入
- 数据级无权限：可进入页面，但当前对象/资源只读

## 19. 设计师出稿注意事项

- 首页不要画成纯说明页，要画成真正的功能入口页
- 双视角要强调“入口不同、数据同源”
- 四类权限页签要统一外壳，避免每页重新发明布局
- 菜单权限与资源权限要重点表达“继承”和“来源”
- 行权限重点表达“规则”和“白名单”
- 列权限重点表达“策略”和“预览”
- 所有页面都要补成功态、失败态、空态、只读态
