# 组织管理一期交付优先 PRD Design

## 1. 文档目标

- 本文档用于沉淀 `组织管理中心 > 组织管理` 模块的一期交付优先 PRD 设计。
- 本文档只覆盖组织管理单模块，不扩展到组织管理中心其他模块。
- 本文档以官方 `DataEase v2 X-Pack > 组织管理` 文档为需求基线，以本次一期交付范围为准。
- 本文档是产品需求设计文档，不进入实现计划，不直接约束编码拆分。

## 2. 设计结论

### 2.1 文档定位

- 文档类型：一期交付优先 PRD
- 需求基线：官方 `v2` 组织管理能力
- 文档主视角：本次一期交付范围
- 非文档目标：
  - 不做 7 模块总览
  - 不做实现计划
  - 不做接口设计文档
  - 不引入新的 Calicat 原型页面

### 2.2 一期交付范围

- 组织列表页
- 左侧组织树与右侧列表联动
- 全局搜索组织名称
- 新增组织
- 新增子组织
- 编辑组织名称
- 删除组织
- 系统管理员与组织管理员的组织范围差异说明

### 2.3 本期非范围

- 组织移动
- 组织排序
- 根组织页面化管理入口
- 根组织页面化改名入口
- 新增独立原型页

### 2.4 官方基线说明

- 本文档仅采用官方 `v2` 组织管理文档中的明确能力作为主基线：
  - 查询组织
  - 新建组织
  - 修改组织名称
  - 删除组织
- 不将其他宣传口径或旧版本口径中的“移动组织/调整上级组织”自动纳入本期需求。

## 3. PRD 结构设计

### 3.1 正文结构

1. 文档目标
2. 产品范围
3. 角色边界与组织模型
4. 页面结构与信息架构
5. 查询与树表联动
6. 新增组织

8. 删除组织
9. 产品规则总表
10. 非范围项
11. 原型节点映射附录

### 3.2 结构原则

- 正文按能力域展开，不按原型页面顺序平铺。
- 附录负责回映 Calicat 节点，不在正文重复原型结构描述。
- 正文只写本期交付，不把旧版本能力和宣传页能力自动拉入范围。
- 以现有原型主体为准，对原型未显式展开但本期必须存在的内容，只补充交互说明、状态说明、校验说明。

## 4. 产品范围设计

### 4.1 范围原则

- 只写本次一期要交付的组织管理能力。
- 以官方 `v2` 组织管理为参考，但不把官方未明确写入的能力自动纳入交付。
- 页面主体严格对齐 Calicat 原型，不额外新增新的页面载体。

### 4.2 成功定义

系统管理员可以在组织管理页完成以下闭环：

- 浏览组织树
- 查看一级组织与下级组织
- 搜索定位组织
- 新增一级组织
- 新增子组织
- 编辑组织名称
- 删除满足条件的组织

组织管理员可以在授权范围内完成同类闭环。

只要以上闭环成立，即视为组织管理一期完成。

## 5. 角色边界与组织模型

### 5.1 使用角色

- 系统管理员
- 组织管理员

### 5.2 页面主视角

- 主流程按系统管理员展开。
- 组织管理员只补限制说明，不单独展开一套完整页面设计。

### 5.3 组织管理员范围规则

- 只能看到并管理自己所在组织及其全部下级组织。
- 只能在该范围内新增、编辑、删除组织。
- 不可查看其他组织分支。
- 全局搜索结果也只能返回其权限范围内的组织。

### 5.4 组织模型

- 系统存在固定根组织。
- 根组织本期不在页面中直接展示。
- 根组织不可删除。
- 根组织本期不提供改名入口。
- 页面展示从一级业务组织开始。

### 5.5 一级组织规则

- 新增组织时，`上级组织` 允许为空。
- 当 `上级组织` 为空时，表示创建一级组织。
- 一级组织本质上挂在系统根组织下，但页面不显式暴露根组织。

## 6. 页面结构与信息架构

### 6.1 页面总体结构

组织管理列表页采用“左树右表”结构：

- 左侧：组织树
- 右侧：页面标题、添加按钮、搜索框、组织列表

### 6.2 首页初始态

- 左侧树不默认选中任何节点。
- 右侧默认展示全部一级组织。

### 6.3 树表联动规则

- 左侧选中某个组织节点后，右侧只展示该节点的直属下级组织。
- 右侧不展示当前选中节点自身。
- 根组织不在左侧树中直接展示。

### 6.4 列表字段

列表严格按原型字段输出：

- 组织名称
- 下属组织数
- 创建时间
- 操作

补充规则：

- `下属组织数` 只统计当前组织的直属子组织数量。
- `创建人` 不在列表展示。
- 但组织创建时默认记录当前操作人员为创建人。

### 6.5 行级操作

操作列保留以下能力：

- 添加子组织
- 编辑组织
- 删除组织

## 7. 查询与树表联动

### 7.1 搜索目标

- 快速定位组织树中的目标组织。

### 7.2 搜索范围

- 搜索范围为全局组织名称搜索。
- 系统管理员搜索全量组织。
- 组织管理员仅搜索自己权限范围内的组织。

### 7.3 搜索联动规则

- 搜索命中后，左侧树展开命中节点的祖先路径。
- 左侧树高亮命中节点。
- 右侧展示命中节点的直属下级组织。

### 7.4 搜索无结果

- 搜索无结果时，左侧不切换当前选中状态。
- 右侧展示无匹配结果提示。

## 8. 新增组织

### 8.1 功能目标

- 新增一级组织或新增当前组织的子组织。

### 8.2 触发入口

- 页面顶部 `添加组织`
- 行级 `添加子组织`

### 8.3 弹窗字段

- 组织名称
- 上级组织

### 8.4 页面顶部新增规则

- 通过“添加组织”弹窗进行新增。
- `上级组织` 可为空。
- `上级组织` 为空时，创建一级组织。
- `上级组织` 非空时，创建为对应组织的子组织。

### 8.5 行级新增下级规则

- 点击行级 `添加子组织` 后，复用“添加组织”弹窗。
- `上级组织` 自动带入当前组织。
- `上级组织` 字段锁定，不允许修改。

### 8.6 成功结果

- 新增成功后关闭弹窗。
- 刷新左侧树与右侧列表。
- 新增一级组织后，出现在一级组织列表中。
- 新增子组织后，出现在对应父组织下。

## 9. 编辑组织

### 9.1 功能目标

- 修改组织名称。

### 9.2 触发入口

- 行级 `编辑组织`

### 9.3 弹窗字段

- 组织名称

### 9.4 编辑规则

- 编辑组织弹窗只允许修改组织名称。
- 不出现 `上级组织` 字段。
- 不承载组织移动能力。

### 9.5 成功结果

- 编辑成功后关闭弹窗。
- 左侧树与右侧列表中的名称即时刷新。
- 页面当前浏览上下文保持不变。

## 10. 删除组织

### 10.1 功能目标

- 删除满足条件的组织。

### 10.2 触发入口

- 行级 `删除组织`

### 10.3 删除入口状态

- 删除按钮保持可点击。
- 即便组织当前不可删除，删除按钮也不置灰。
- 用户点击后，根据阻断原因给出明确反馈。

### 10.4 可删除场景

- 当组织不存在子组织、用户、资源时，可进入删除确认弹窗。
- 用户确认后执行删除。
- 删除成功后刷新树与列表。

### 10.5 禁删场景

任一条件成立，即禁止删除：

- 存在子组织
- 存在用户
- 存在资源

### 10.6 禁删反馈

禁删提示按原因分别给出不同文案：

- 存在子组织：提示需先删除子组织
- 存在用户：提示需先处理该组织下用户
- 存在资源：提示需先处理该组织下资源

本期不提供以下能力：

- 强制删除
- 迁移资源后再删
- 放弃资源直接删

## 11. 表单与校验规则

### 11.1 组织名称校验

- 必填
- 长度 `1-20`
- 只允许中文、英文、数字
- 不允许空格
- 不允许任何符号或特殊字符

### 11.2 重名校验

- 同一上级组织下名称不能重复
- 不同分支允许同名

### 11.3 校验失败结果

- 表单内给出明确错误提示
- 不关闭弹窗
- 不提交保存

## 12. 页面状态与异常场景

### 12.1 页面状态

- 初始态：
  - 左树不选中
  - 右侧展示全部一级组织
- 节点选中态：
  - 右侧展示当前节点直属下级
- 搜索命中态：
  - 左树展开路径并高亮命中节点
  - 右侧展示命中节点直属下级
- 空列表态：
  - 当前节点无直属子组织时展示空状态
- 新增成功态：
  - 树和列表即时刷新
- 编辑成功态：
  - 名称即时更新
- 删除成功态：
  - 被删组织从树和列表中移除

### 12.2 异常场景

- 名称为空
- 名称超长
- 名称包含空格、符号或特殊字符
- 同一上级下名称重复
- 删除时存在子组织
- 删除时存在用户
- 删除时存在资源
- 搜索无结果
- 组织管理员尝试查看或操作越权组织

## 13. 产品规则总表

| 主题 | 规则 |
| --- | --- |
| 页面主体 | 严格以现有 Calicat 原型主体为准，不新增原型页 |
| 官方基线 | 只采用官方 `v2` 文档中明确写出的查询/新建/改名/删除能力 |
| 根组织 | 固定存在、页面不展示、不可删除、本期无改名入口 |
| 首页初始态 | 左树不选中，右侧展示全部一级组织 |
| 树表联动 | 左侧选中节点，右侧展示直属下级组织 |
| 搜索 | 全局按名称搜索，命中后展开祖先路径并高亮命中节点 |
| 新增组织 | 页面级新增可空上级组织，空值表示一级组织 |
| 添加子组织 | 复用新增弹窗，上级组织自动带入且锁定 |
| 编辑组织 | 只改名称，不改上级组织 |
| 删除组织 | 存在子组织、用户、资源任一项即禁止删除 |
| 下属组织数 | 只统计直属子组织数量 |
| 创建人 | 默认记录当前操作人员，但不在列表展示 |
| 名称规则 | 1-20 字符，仅中文/英文/数字，不允许空格和特殊字符 |
| 重名规则 | 同一上级下不可重名，不同分支可同名 |
| 组织管理员 | 仅可管理自己所在组织及全部下级组织 |

## 14. 非范围项

- 组织移动
- 组织排序
- 页面化根组织管理
- 页面化根组织改名
- 新增新的原型页或专门的高级设置面板

## 15. 原型节点映射附录

### 15.1 页面映射

- 组织管理列表页：承接页面结构、组织树、列表字段、操作列、搜索框
- 添加组织弹窗：承接新增一级组织与新增子组织
- 编辑组织弹窗：承接组织名称编辑
- 删除确认弹窗：承接可删除场景下的确认动作

### 15.2 原型与规则补充关系

- 添加组织弹窗原型已体现 `组织名称 + 上级组织`
- 编辑组织弹窗原型只体现 `组织名称`
- 行级 `添加子组织` 通过复用添加组织弹窗完成，不新增新弹窗原型
- 删除禁删场景的多原因文案属于规则补充，不新增原型页
