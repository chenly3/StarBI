# 用户管理详细需求分析

## 1. 模块基线

### 1.1 模块定位

- 用户管理是组织管理中心中负责“谁能进入系统、在什么组织下、以什么角色工作”的主控模块。
- 它同时承接账号生命周期、组织归属、角色归属、启停状态、导入扩容等职责。

### 1.2 目标用户

- 系统管理员
- 组织管理员
- 被管理对象：组织成员、外部待纳入成员、角色持有人

### 1.3 关键对象

- 用户
- 组织
- 角色
- 用户来源
- 启用状态
- 默认密码/初始密码

### 1.4 总体业务规则

- 用户必须运行在某个组织上下文内。
- 账号创建后不可修改。
- 用户可在当前组织下拥有多个角色。
- 组织之间相互隔离，上下级组织也不继承资源。
- 菜单权限绑定角色，资源权限可绑定用户或角色，因此角色设计先于权限可用性。
- 自定义角色只能继承组织内置角色，权限不能超过父角色。

### 1.5 当前现状总览

- 已有：
  - 官方 X-Pack 文档明确功能边界
  - Calicat 已完成用户管理核心页面原型
  - 仓库已有 API 包装层、后端接口合同、语言包文案
- 缺失：
  - 本地前端页面
  - 本地后端 X-Pack 业务实现
  - `de-xpack` 本地落地代码
- 结论：
  - 当前阶段适合先完成单模块需求基线、原型对齐和实施切片，不适合直接假设已有完整代码底座

## 2. 用户列表

### 2.1 功能目标

- 在当前组织上下文中查看、筛选、搜索、分页管理用户。

### 2.2 使用角色

- 系统管理员
- 组织管理员

### 2.3 前置条件

- 当前用户拥有组织管理中心入口与用户管理菜单权限。
- 已经进入某个组织上下文。

### 2.4 触发入口

- 左侧导航 `用户管理`
- 页面页签默认落在“用户”

### 2.5 页面元素

- 组织切换入口
- 搜索框
- 筛选项：用户来源、启用状态、角色
- 表格
- 行级操作
- 批量操作
- 分页器
- `添加用户` / `批量导入` 按钮

### 2.6 字段说明

- 表格字段建议以以下口径为准：
  - 姓名
  - 账号
  - 角色
  - 邮箱
  - 用户来源
  - 启用状态
  - 创建时间
- 说明：
  - 原型出现了“所属组织”列
  - 但当前仓库 `UserGridVO` 不返回组织字段，因此该列现在只能视为“官方/原型基线字段”，不能视为现有合同字段

### 2.7 操作流程

1. 进入用户列表页。
2. 如有需要切换组织上下文。
3. 输入关键字或设置筛选条件。
4. 查看列表并执行新增、编辑、重置密码、删除、启停、批量操作。

### 2.8 关键业务规则

- 列表是“当前组织视角”，不是全系统扁平用户池。
- 系统管理员与组织管理员都可管理本组织用户。
- 筛选条件至少支持 `origin`、`enable`、`roleId`，与 `UserGridRequest` 一致。

### 2.9 校验规则

- 关键字搜索支持姓名、账号、邮箱。
- 分页和排序默认可按创建时间倒序。

### 2.10 异常场景

- 无权限进入
- 搜索无结果
- 当前组织暂无用户
- 切换启用状态失败

### 2.11 依赖模块

- 组织管理
- 权限配置

### 2.12 当前现状

- 已有：
  - `userPageApi -> /user/pager/{page}/{limit}`
  - `UserGridRequest` 已定义筛选口径
  - 原型已覆盖列表、筛选、分页、批量操作
- 缺失：
  - 本地页面实现
- 待确认：
  - 是否支持默认“时间倒序”切换控件直出

### 2.13 复刻基线

- 按官方文档与原型 1:1 复刻当前组织视角下的用户列表管理页。

### 2.14 适配建议

- 一期直接采用“组织切换 + 当前组织列表”，不要额外引入“所属组织筛选”打乱组织隔离模型。
- 列表先支持单层筛选，不先上复杂高级筛选。

### 2.15 产品执行计划

- 冻结列表字段和筛选口径。
- 明确系统管理员与组织管理员的可见范围差异。
- 对齐行级操作顺序与批量操作范围。
- 确认空状态、无结果、无权限三类状态文案。

## 3. 添加用户

### 3.1 功能目标

- 在当前组织下新增一个本地用户并分配角色。

### 3.2 使用角色

- 系统管理员
- 组织管理员

### 3.3 前置条件

- 已切换到目标组织。
- 当前组织已存在可分配角色。

### 3.4 触发入口

- 用户列表页 `添加用户`

### 3.5 页面元素

- 顶部提示条
- 账号
- 姓名
- 邮箱
- 手机区号 + 手机号
- 角色多选
- 是否启用
- 取消 / 确定

### 3.6 字段说明

- 账号：唯一标识，创建后不可修改
- 姓名：展示名称
- 邮箱：通知与身份信息
- 手机：手机号信息
- 角色：当前组织内角色，可多选
- 是否启用：禁用后不可登录

### 3.7 操作流程

1. 在目标组织进入新增弹窗。
2. 填写基础信息。
3. 选择当前组织可分配角色。
4. 设置启用状态。
5. 提交并返回列表刷新。

### 3.8 关键业务规则

- 新增动作发生在“当前组织”内，API `UserCreator` 没有 `orgId`，说明组织来自上下文。
- 用户在当前组织的角色可多选。
- 用户来源不是新增表单手工字段，更像系统派生字段。

### 3.9 校验规则

- 账号、姓名、邮箱、角色、启用状态必填
- 账号匹配现有账号格式规则
- 邮箱与手机号格式合法

### 3.10 异常场景

- 当前组织无可分配角色
- 账号重复
- 邮箱格式错误
- 提交失败

### 3.11 依赖模块

- 组织管理
- 角色管理

### 3.12 当前现状

- 已有：
  - `userCreateApi -> /user/create`
  - `UserCreator` 字段已定义
  - 原型已覆盖全部核心字段
- 缺失：
  - 本地弹窗实现
  - 前端未提供 `createPlatform` API 包装
- 待确认：
  - 系统管理员是否需要显式展示“当前组织”说明

### 3.13 复刻基线

- 按官方文档复刻“先选组织，再新增”的模式。

### 3.14 适配建议

- 不在一期新增“所属组织”字段，避免与当前接口模型冲突。
- 顶部提示条明确“当前用户将创建在所选组织下”。

### 3.15 产品执行计划

- 确认组织上下文展示方式。
- 确认角色多选交互与默认值。
- 明确账号重复、表单校验失败、创建成功提示。

## 4. 批量导入用户

### 4.1 功能目标

- 通过 Excel 批量向当前组织导入用户，并支持部分成功处理。

### 4.2 使用角色

- 系统管理员
- 组织管理员

### 4.3 前置条件

- 已进入目标组织。
- 已下载并使用标准模板。

### 4.4 触发入口

- 用户列表页 `批量导入`

### 4.5 页面元素

- 模板下载提示
- 文件上传区
- 文件格式/大小限制提示
- 导入按钮
- 导入结果弹窗
- 错误报告下载入口

### 4.6 字段说明

- 模板字段至少覆盖新增用户所需核心字段。
- 第三方平台用户需通过 `唯一标识` 对应外部平台账户。

### 4.7 操作流程

1. 下载模板。
2. 填写用户数据并上传。
3. 后台校验。
4. 成功数据创建，不合规数据进入错误报告。
5. 若部分成功，用户下载错误报告并修正后再次导入。

### 4.8 关键业务规则

- 部分成功允许落库。
- 错误数据必须可追溯下载。
- 文件大小上限 `10M`。
- 用户来源支持本地与第三方来源枚举。

### 4.9 校验规则

- 只支持 `.xlsx` / `.xls`
- 一次只上传一个文件
- 模板格式错误需直接阻断

### 4.10 异常场景

- 模板错误
- 超过大小限制
- 批量数据部分失败
- 错误报告下载失败

### 4.11 依赖模块

- 用户管理
- 第三方登录/用户绑定能力

### 4.12 当前现状

- 已有：
  - `downExcelTemplateApi -> /user/excelTemplate`
  - `importUserApi -> /user/batchImport`
  - `downErrorRecordApi -> /user/errorRecord/{key}`
  - `UserApi.createPlatform` / `PlatformUserCreator` 说明后端合同同时考虑了第三方来源用户链路
  - `UserImportVO` 已定义成功/失败计数
  - 原型已覆盖导入弹窗与结果弹窗
- 缺失：
  - 本地页面实现
- 待确认：
  - 模板具体列与错误报告样式

### 4.13 复刻基线

- 复刻“导入成功一部分，失败一部分”的后台导入范式。

### 4.14 适配建议

- 一期保留摘要式结果弹窗，不额外在前端渲染错误表格。
- 错误报告以下载为主，避免前端承担过多解析逻辑。

### 4.15 产品执行计划

- 冻结导入模板字段。
- 冻结来源枚举和第三方唯一标识说明。
- 明确部分成功结果反馈与错误报告保留策略。

## 5. 编辑用户

### 5.1 功能目标

- 修改用户基础信息、角色与启用状态，但不允许修改账号。

### 5.2 使用角色

- 系统管理员
- 组织管理员

### 5.3 前置条件

- 用户已存在。

### 5.4 触发入口

- 用户列表行级 `编辑`

### 5.5 页面元素

- 与新增相同的表单结构
- 账号字段只读

### 5.6 字段说明

- 除账号外，其余基础字段可编辑。

### 5.7 操作流程

1. 从列表打开编辑弹窗。
2. 回填用户信息。
3. 修改角色或状态。
4. 提交保存。

### 5.8 关键业务规则

- 官方明确账号不可修改。
- 当前组织内角色允许调整。

### 5.9 校验规则

- 与新增一致
- 账号禁用编辑

### 5.10 异常场景

- 详情查询失败
- 保存失败

### 5.11 依赖模块

- 用户详情查询
- 角色选项查询

### 5.12 当前现状

- 已有：
  - `queryFormApi -> /user/queryById/{uid}`
  - `userEditApi -> /user/edit`
  - `UserFormVO` / `UserEditor`
- 缺失：
  - 本地弹窗实现

### 5.13 复刻基线

- 复刻新增弹窗布局并将账号设为只读。

### 5.14 适配建议

- 新增与编辑共用同一表单组件。

### 5.15 产品执行计划

- 冻结回填字段与只读规则。
- 明确编辑后列表刷新策略。

## 6. 重置密码

### 6.1 功能目标

- 将目标用户密码重置为系统初始密码。

### 6.2 使用角色

- 系统管理员
- 组织管理员

### 6.3 前置条件

- 目标用户存在。

### 6.4 触发入口

- 用户列表行级 `重置密码`

### 6.5 页面元素

- 二次确认弹窗
- 默认密码提示文案

### 6.6 字段说明

- 无表单字段，只有确认信息。

### 6.7 操作流程

1. 点击重置密码。
2. 系统弹出确认。
3. 确认后调用重置接口。
4. 返回成功提示。

### 6.8 关键业务规则

- 重置目标是“初始密码”，不是自定义输入新密码。

### 6.9 校验规则

- 仅具备管理权限的用户可操作。

### 6.10 异常场景

- 当前用户无权限
- 重置失败

### 6.11 依赖模块

- 系统参数中的默认密码规则

### 6.12 当前现状

- 已有：
  - `defaultPwdApi -> /user/defaultPwd`
  - `resetPwdApi -> /user/resetPwd/{uid}`
  - locale 已有确认与成功文案
- 缺失：
  - 本地确认弹窗实现
- 待确认：
  - 是否在确认弹窗中展示默认密码明文

### 6.13 复刻基线

- 按官方文档复刻“恢复为初始密码”。

### 6.14 适配建议

- 一期只提示“恢复为初始密码”，不直接展示密码值。

### 6.15 产品执行计划

- 明确默认密码展示策略。
- 明确审计日志是否纳入后续范围。

## 11. 角色移除用户

### 11.1 功能目标

- 将用户从当前角色移除，并明确影响范围。

### 11.2 使用角色

- 系统管理员
- 组织管理员

### 11.3 前置条件

- 用户已挂载在当前角色下。

### 11.4 触发入口

- 角色成员表行级 `移除`

### 11.5 页面元素

- 二次确认弹窗
- 影响提示文案

### 11.6 字段说明

- 无表单字段。

### 11.7 操作流程

1. 点击移除。
2. 先调用影响预判接口。
3. 根据返回结果展示不同风险提示。
4. 确认后解绑。

### 11.8 关键业务规则

- 如果用户仍有当前组织其他角色，只移出本角色。
- 如果用户在当前组织无其他角色，但在其他组织仍有角色，则从当前组织移除。
- 如果该角色是用户在系统的唯一角色，则用户从系统删除。

### 11.9 校验规则

- 先预判，再解绑。

### 11.10 异常场景

- 预判失败
- 解绑失败

### 11.11 依赖模块

- 角色管理
- 组织管理

### 11.12 当前现状

- 已有：
  - `beforeUnmountInfoApi -> /role/beforeUnmountInfo`
  - `unMountUserApi -> /role/unMountUser`
  - locale 已有 `clear_in_system` / `clear_in_org`
  - 官方文档明确唯一角色删除规则
- 缺失：
  - 本地确认弹窗实现

### 11.13 复刻基线

- 复刻官方“先提示影响，再移除”的流程。

### 11.14 适配建议

- 一期必须把 3 种影响结果分别提示，不能只给统一删除确认。

### 11.15 产品执行计划

- 冻结 `beforeUnmountInfo` 返回值与前端文案映射。
- 明确成功后的列表刷新与角色人数刷新策略。

## 13. 单模块结论

### 13.1 需求边界

- 本模块不是单纯的“用户 CRUD”。
- 它同时是组织隔离、角色建模、权限落地、第三方用户接入的入口层。

### 13.2 实施优先级

- P0：
  - 用户列表
  - 添加/编辑用户
  - 查询用户
  - 删除用户
  - 启停切换
  - 角色列表
  - 角色添加/移除用户
  - 创建自定义角色
- P1：
  - 批量导入第三方来源细节
  - 第三方平台直接创建用户流
  - 默认密码展示策略
  - 外部用户原型补齐

### 13.3 当前最重要的 3 个决策点

- 组织归属是否继续采用“隐式当前组织”模式。
- 外部用户添加是否一期落地。
- 角色移除用户的三类影响提示如何产品化呈现。
