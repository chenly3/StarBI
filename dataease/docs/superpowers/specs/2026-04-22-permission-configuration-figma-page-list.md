## 3. Figma 页面结构建议

### 3.1 页面分组

建议在 Figma 中按以下 Page 或 Section 分组：

- `A. 权限配置-首页与入口`
- `B. 权限配置-双视角框架`
- `C. 权限配置-四类权限页`
- `D. 权限配置-弹窗`
- `E. 权限配置-状态页`

### 3.2 命名规范

页面或 Frame 建议统一采用以下格式：

- `PC / 权限配置 / 首页 / 正常态`
- `PC / 权限配置 / 按用户配置 / 未选对象`
- `PC / 权限配置 / 菜单权限 / 按用户视角 / 正常态`
- `PC / 权限配置 / 行权限 / 白名单弹窗 / 正常态`

这样方便后续按：

- 端
- 模块
- 页面
- 场景
- 状态

进行检索和评审。

## 4. 全局组件清单

### 4.1 必须复用的全局组件

以下组件建议做成共享组件或共享样式：

- 页面标题栏
- 当前组织信息条
- 视角切换组件
- 对象/资源搜索框
- 左侧树/列表容器
- 页签组件
- 保存操作栏
- 状态提示条
- 空状态卡片
- 无权限卡片
- 只读提示条
- 成功反馈 Toast 样式
- 未保存离开确认弹窗

### 4.2 全局组件状态

以下组件至少要做这些状态：

- 默认
- Hover
- Selected
- Disabled
- Readonly
- Loading
- Error

### 4.3 统一视觉规则

为了降低信息密度带来的阅读成本，建议统一以下规则：

- 左侧承载对象或资源选择，右侧承载配置内容
- 页签始终放在配置内容区顶部
- 保存区始终固定在内容区底部或页面底部
- 当前组织和当前视角始终可见
- 继承状态和只读状态使用固定标签体系

## 16. 设计交付顺序建议

建议设计师按以下顺序出稿：

1. 首页
2. 双视角骨架页
3. 菜单权限页
4. 资源权限页
5. 行权限页
6. 列权限页
7. 白名单弹窗
8. 脱敏规则弹窗
9. 通用状态页

这样可以先锁定主框架，再补复杂页。

## 17. 评审清单

在 Figma 评审时，逐页核对以下问题：

- 当前组织是否持续可见
- 当前视角是否明确
- 当前对象或资源是否明确
- 直接授权与继承授权是否可区分
- 保存按钮是否明确
- 未保存离开是否有提示
- 角色差异是否表达
- 空态、只读态、无权限态是否完整
