# 数据集作用域数据权限设计

## 1. 文档目标

本文档用于收口 `StarBI / DataEase 二开项目` 中 `行权限 / 列权限` 的最新产品归属、入口结构和页面边界。

本文档只解决一件事：

- 把原先被放在 `权限配置` 模块中的 `行权限 / 列权限`，重定义为 `问数配置 -> 问数资源管理 -> 具体数据集` 下的数据权限能力

本文档不重新设计已确认的静态页面主体布局，只定义：

- 信息架构
- 页面入口
- 数据边界
- 页面上下文
- 交互边界
- 模块职责拆分

## 2. 背景与问题

在此前的权限配置 PRD 中，`菜单权限 / 资源权限 / 行权限 / 列权限` 被统一放在 `系统设置 -> 权限配置` 中讨论。

但本轮确认后，产品边界已经明确变化：

- `行权限 / 列权限` 本质上都作用于某一张具体数据集
- 这两类能力不应脱离数据集上下文独立配置
- 入口应从 `问数配置 -> 问数资源管理` 的数据集出发
- 已经完成并确认过的 `行权限页 / 列权限页` 静态原型主体不再重做布局

因此，需要新增一份收口文档，把 `数据权限属于数据集` 的新口径写死，供后续研发和文档同步使用。

## 3. 输入依据

### 3.1 已确认的用户决策

本次对话已确认以下边界：

- `系统设置 -> 权限配置` 只保留 `菜单权限 / 资源权限`
- `行权限 / 列权限` 不再作为 `权限配置` 模块中的独立全局页面
- `行权限 / 列权限` 迁移到 `系统设置 -> 问数配置 -> 问数资源管理`
- 入口放在数据集列表每一行最后一列 `...` 操作菜单中
- 点击 `行权限配置 / 列权限配置` 后进入独立整页
- 不采用页签入口
- 仅对当前仓库实际可支持的数据集显示入口
- 对不支持的数据集，操作菜单中不显示 `行权限配置 / 列权限配置`
- 已确认过的 `行权限 / 列权限` 静态页主体继续复用，只补：
  - `返回问数资源管理`
  - `当前数据集名称`
- 页面主体不重新设计布局

## 4. 核心设计结论

### 4.1 模块职责拆分

本期权限相关能力按以下方式拆分：

- `系统设置 -> 权限配置`
  - 只保留：`菜单权限`、`资源权限`
- `系统设置 -> 问数配置 -> 问数资源管理`
  - 承载：`行权限`、`列权限`

职责口径如下：

- `权限配置` 负责“谁能访问什么功能、什么资源”
- `问数配置` 负责“具体数据集的数据访问控制”

### 4.2 数据权限的产品定义

本期统一将 `行权限 / 列权限` 定义为：

- 某一张具体数据集的数据权限配置能力

其中：

- `行权限` 定义当前数据集哪些数据行对哪些对象可见
- `列权限` 定义当前数据集哪些字段正常显示、禁用或脱敏

它们不再被定义为系统级独立权限模块。

### 4.3 页面复用原则

已确认过的 `行权限页 / 列权限页` 静态原型主体继续复用。

本期仅允许新增以下上下文元素：

- `返回问数资源管理`
- `当前数据集名称`

本期不允许：

- 重新设计行权限页主体布局
- 重新设计列权限页主体布局
- 在未再次确认的前提下改动已确认的静态页结构

## 5. 信息架构与页面关系

### 5.1 目标结构

调整后的系统设置结构如下：

- `系统设置`
  - `权限配置`
    - `菜单权限`
    - `资源权限`
  - `问数配置`
    - `问数资源管理`
      - 数据集列表
      - 数据集行尾 `...`
        - `行权限配置`
        - `列权限配置`

### 5.2 页面跳转关系

标准路径如下：

1. 用户进入 `系统设置 -> 问数配置 -> 问数资源管理`
2. 在某个数据集行尾点击 `...`
3. 选择 `行权限配置` 或 `列权限配置`
4. 进入对应独立整页
5. 页面头部显示：
   - `返回问数资源管理`
   - `当前数据集名称`
6. 用户完成配置后保存或返回列表

### 5.3 不再采用的结构

本期明确不采用以下结构：

- 在 `权限配置` 模块中继续保留 `行权限 / 列权限` 独立页
- 在 `权限配置` 中点击后再跳去数据集上下文
- 在数据集详情中采用页签切换 `行权限 / 列权限`

## 6. 数据边界与入口显示规则

### 6.1 数据集支持边界

并非所有数据集都天然支持行权限和列权限。

本期采用“按现有仓库实际能力显示”的原则：

- 只有系统运行时能够取得字段模型、查询模型或权限配置所需元数据的数据集，才认为“支持数据权限”
- 当前仓库不具备配置前提的数据集，不提供行权限和列权限入口

### 6.2 列表入口显示规则

在 `问数资源管理` 的数据集列表中：

- 对支持的数据集：
  - 行尾 `...` 菜单显示 `行权限配置`
  - 行尾 `...` 菜单显示 `列权限配置`
- 对不支持的数据集：
  - 行尾 `...` 菜单中不显示这两个菜单项

本期不采用以下方式：

- 菜单项置灰
- 新增“是否支持数据权限”列
- 额外展示说明型弱提示文案

### 6.3 页面上下文规则

进入行权限页或列权限页后：

- 页面只作用于当前这一张数据集
- 页面头部固定显示 `当前数据集名称`
- 页面提供统一返回入口 `返回问数资源管理`
- 页面内不允许再次切换其他数据集

如果要配置其他数据集，必须：

1. 返回问数资源管理列表
2. 从对应数据集行尾操作重新进入

## 7. 行权限页面交互边界

### 7.1 页面定位

行权限页是“当前数据集的行级权限配置页”，不是全局规则页，也不是跨数据集规则维护页。

### 7.2 保留能力

页面继续保留已确认的原型能力：

- 配置对象包括：
  - 角色
  - 用户
  - 系统变量
- 可执行操作包括：
  - 新增规则
  - 编辑规则
  - 删除规则
  - 配置白名单
  - 显式保存

### 7.3 明确限制

页面内不允许：

- 切换到其他数据集
- 在当前页中维护跨数据集通用模板
- 把白名单抽成独立于规则之外的单独模块

白名单继续定义为：

- 某一条行权限规则的附属配置

### 7.4 保存与离开

页面统一采用显式保存。

当存在未保存修改时，以下动作都需要离开确认：

- 返回列表
- 路由跳转
- 页面刷新
- 关闭页面

保存失败时：

- 保留当前输入
- 不清空用户已编辑内容

## 8. 列权限页面交互边界

### 8.1 页面定位

列权限页是“当前数据集的字段级权限配置页”。

### 8.2 保留能力

页面继续保留已确认的原型能力：

- 展示当前数据集下可识别字段
- 配置字段策略：
  - 正常
  - 禁用
  - 脱敏
- 打开脱敏规则弹窗
- 选择系统内置脱敏规则
- 配置自定义脱敏规则
- 预览脱敏效果
- 显式保存

### 8.3 明确限制

页面内不允许：

- 跨数据集选择字段
- 从当前页直接切换到另一张数据集继续配置
- 把字段建模、字段编辑等数据建模能力并入列权限页

### 8.4 保存与离开

列权限页与行权限页保持一致：

- 显式保存
- 未保存离开确认
- 保存失败时保留当前编辑状态

## 9. 异常态与无效上下文处理

### 9.1 入口隐藏态

对于不支持数据权限的数据集：

- 列表页不展示 `行权限配置 / 列权限配置`

这是主路径处理方式。

### 9.2 直达异常态

如果用户通过缓存路由、非法链接或运行时状态变化进入页面，但系统发现当前数据集已不满足配置前提，则页面不进入可编辑态。

此时页面应显示：

- 当前数据集暂不支持行权限配置

或：

- 当前数据集暂不支持列权限配置

并提供：

- `返回问数资源管理`

### 9.3 上下文失效态

当出现以下情况时：

- 数据集已删除
- 当前用户已无权限访问该数据集
- 当前组织上下文已失效

页面应显示明确的无效上下文提示，不保留误导性的空白编辑区域。

## 10. 模块口径与用户认知

### 10.1 权限配置模块的用户认知

用户进入 `系统设置 -> 权限配置` 时，应理解为：

- 我在配置角色或用户能否访问某菜单
- 我在配置角色或用户能否操作某资源

### 10.2 问数配置模块的用户认知

用户进入 `系统设置 -> 问数配置 -> 问数资源管理` 时，应理解为：

- 我在管理具体问数数据集
- 我可以基于某张具体数据集继续配置其数据可见范围和字段可见策略

### 10.3 统一口径

后续所有 PRD、交互稿、Figma 标注和研发说明需要统一改成以下口径：

- `权限配置` 不再包含 `行权限 / 列权限`
- `行权限 / 列权限` 归属于 `问数配置 -> 数据集操作`
- 凡是提到“数据权限”，默认指向“具体数据集上下文下的行/列权限”

## 11. 本期范围与非范围

### 11.1 本期范围

本期范围包括：

- 调整 `行权限 / 列权限` 的模块归属
- 调整 `问数资源管理` 列表行尾操作入口
- 复用已确认的行权限页和列权限页主体
- 在页面头部补 `返回问数资源管理 + 当前数据集名称`
- 明确支持/不支持数据集的入口显示规则

### 11.2 本期非范围

本期不包括：

- 重画行权限页主体布局
- 重画列权限页主体布局
- 在页内增加数据集切换器
- 设计新的“数据权限总览页”
- 给不支持的数据集新增单独的支持状态说明列
- 在 `权限配置` 中保留指向数据权限的跳转入口

## 12. 验收标准

当以下条件同时满足时，视为本轮设计落地正确：

1. `系统设置 -> 权限配置` 中只保留 `菜单权限 / 资源权限`
2. `行权限 / 列权限` 的产品文案和页面归属全部迁移至 `问数配置`
3. `问数资源管理` 数据集列表行尾 `...` 按支持能力显示 `行权限配置 / 列权限配置`
4. 点击入口后进入已确认的静态整页，而不是新布局
5. 页面头部明确显示 `返回问数资源管理 + 当前数据集名称`
6. 页面内不能跨数据集切换
7. 不支持的数据集不展示入口
8. 通过非法入口进入时，页面能正确显示“不支持或无效上下文”提示
