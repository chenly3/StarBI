# 行列权限配置系统菜单设计

日期：2026-04-24

## 1. 文档目标

本文档用于收口 `StarBI / DataEase 二开项目` 中 `行列权限配置` 的本期需求、交互边界和实施口径。

本文档是当前最新口径，替代以下两类历史临时方案：

- 将行列权限挂在 `问数配置 -> 问数资源管理` 数据集行操作中的方案
- 将行列权限作为 `权限配置` 页内第 3、第 4 个页签展示的方案

本期最新结论：

- `行列权限配置` 作为 `系统设置` 左侧一级同级菜单
- 菜单层级与 `权限配置`、`问数配置`、`用户管理`、`组织管理` 同级
- 页面主体使用已确认的 5 个行列权限静态原型壳
- 本期接入真实数据集、行权限、列权限、角色、用户、字段和白名单接口
- 不从 `问数配置` 的数据集操作菜单提供快捷入口

## 2. 输入依据

### 2.1 官方能力依据

DataEase 官方 X-Pack 权限配置文档：

- https://dataease.cn/docs/v2/xpack/sys_management_permission/

官方文档中的关键能力口径：

- 权限配置包含菜单权限、资源权限、数据权限等能力
- 行权限作用于数据集的数据行访问控制
- 列权限作用于数据集字段的禁用、脱敏等控制
- 行权限支持角色、用户、系统变量等配置对象
- 列权限支持字段级禁用、脱敏规则和白名单

### 2.2 本地静态原型依据

本期直接使用以下 5 个已确认静态壳：

- 行权限静态页：`http://localhost:8080/#/prototype/permission/row`
- 列权限静态页：`http://localhost:8080/#/prototype/permission/column`
- 行权限弹窗：`http://localhost:8080/#/prototype/permission/row?dialog=role`
- 列权限弹窗：`http://localhost:8080/#/prototype/permission/column?dialog=column`
- 脱敏规则弹窗：`http://localhost:8080/#/prototype/permission/column?dialog=column&mask=1`

对应源码：

- `de-xpack/permission-management/src/menu/system/permission/RowPermissionPreview.vue`
- `de-xpack/permission-management/src/menu/system/permission/ColumnPermissionPreview.vue`
- `de-xpack/permission-management/src/menu/system/permission/components/RowPermissionDialog.vue`
- `de-xpack/permission-management/src/menu/system/permission/components/ColumnPermissionDialog.vue`
- `de-xpack/permission-management/src/menu/system/permission/components/MaskRuleDialog.vue`

### 2.3 本地接口和模型依据

当前仓库已有或已封装的前端接口包括：

- `listRowPermissions(datasetId, page, size)`
- `listColumnPermissions(datasetId, page, size)`
- `saveRowPermission(data)`
- `saveColumnPermission(data)`
- `deleteRowPermission(data)`
- `deleteColumnPermission(data)`
- `whiteListUsersForPermissions(data)`
- `listFieldsWithPermissions(datasetId)`
- `rowPermissionTargetObjList(...)`

当前仓库已有类型包括：

- `DatasetRowPermissionDTO`
- `DatasetColumnPermissionDTO`
- `ColumnPermissionFieldRule`
- `DatasetPermissionWhiteUser`
- `DatasetPermissionTargetType`

## 3. 本期范围

### 3.1 范围内

本期交付以下能力：

- 新增 `系统设置 -> 行列权限配置` 左侧菜单
- 菜单位置放在 `权限配置` 后、`问数配置` 前
- 新增独立权限点：`行列权限配置`
- 页面路由：`/sys-setting/row-column-permission`
- 页面支持 query 初始化：
  - `/sys-setting/row-column-permission?datasetId=xxx&tab=row`
  - `/sys-setting/row-column-permission?datasetId=xxx&tab=column`
- 页面保留 `返回问数资源管理` 按钮
- 左侧接真实 DataEase 数据集目录树
- 右侧接真实行权限、列权限规则列表
- 行权限新增、编辑、删除、分页、筛选、搜索
- 列权限新增、编辑、删除、分页、筛选、搜索
- 行权限弹窗接真实角色、用户、系统变量、字段、白名单数据
- 列权限弹窗接真实角色、用户、字段、白名单数据
- 脱敏规则弹窗作为列权限弹窗内子弹窗
- 成功、失败、删除确认、空状态按现有 DataEase 消息机制处理

### 3.2 范围外

本期不做以下内容：

- 不在 `问数配置` 数据集行操作菜单中保留行列权限入口
- 不做数据集操作菜单到本页的快捷跳转
- 不绕过登录鉴权或系统权限点鉴权
- 不新增页面级 `保存全部`
- 不新增列权限外层列表的 `字段规则摘要` 列
- 不新增说明型大段文案布局
- 不重构已确认静态原型壳的主体布局
- 不影响 `权限配置` 的按用户配置、按资源配置能力
- 不影响 `问数配置` 当前开发中的数据集操作菜单

## 4. 信息架构

### 4.1 系统设置左侧菜单

调整后的系统设置菜单层级：

- `系统设置`
- `权限配置`
- `行列权限配置`
- `问数配置`
- `用户管理`
- `组织管理`
- `通用配置`
- `字体管理`

具体排序以现有系统设置菜单实际排序为准，但 `行列权限配置` 必须与上述模块同级，不能嵌入 `权限配置` 页签，也不能嵌入 `问数配置`。

### 4.2 页面路由

正式路由：

- `/sys-setting/row-column-permission`

支持 query：

- `datasetId`：指定初始选中的数据集
- `tab`：指定初始页签，合法值为 `row`、`column`

query 只用于页面初始化状态，不代表免登录、不代表免权限校验。

### 4.3 静态预览路由

以下路由继续保留为开发和验收对照，不进入正式菜单：

- `/prototype/permission/row`
- `/prototype/permission/column`
- `/prototype/permission/row?dialog=role`
- `/prototype/permission/column?dialog=column`
- `/prototype/permission/column?dialog=column&mask=1`

## 5. 页面结构

### 5.1 整体布局

正式页面沿用已确认静态壳的主体结构：

- 顶部为当前数据集基础信息和操作区
- 左侧为数据集目录树
- 右侧为规则配置区
- 规则配置区内有 `行权限`、`列权限` 两个同级页签
- 页签内容沿用当前 5 个静态壳的视觉密度、字段和弹窗结构

页面需要纳入系统设置统一自适应规范：

- 不再出现高分辨率下页面主体过窄的问题
- 字体、表格密度、边距、边界与系统设置其他页面保持统一
- 不新增无意义说明块占用空间

### 5.2 左侧数据集树

左侧展示 DataEase 原生数据集目录树：

- 包含文件夹、分组和数据集叶子节点
- 只有数据集叶子节点可选中
- 选中文件夹或分组不加载规则
- 数据集树搜索走后端接口
- 清空搜索后恢复完整树
- 不额外设计分页和懒加载，复用现有数据集树能力

### 5.3 数据集默认选中

进入页面时：

- URL 不带 `datasetId`：自动选中第一个可配置数据集，默认进入 `行权限`
- URL 带有效 `datasetId`：定位并选中该数据集
- URL 带无效或不可见 `datasetId`：回退第一个可配置数据集，并展示错误提示
- 无任何可配置数据集：保留页面空壳，不额外增加说明块

### 5.4 返回问数资源管理

页面保留 `返回问数资源管理` 按钮。

点击后跳转：

- `/sys-setting/query-config?tab=query_resources`

该按钮只做返回，不负责保存当前未提交表单。

## 6. 行权限设计

### 6.1 列表

行权限列表展示规则：

- 当前列表只展示当前选中数据集的行权限规则
- 默认按接口返回顺序展示
- 支持分页
- 支持对象类型筛选
- 支持受限对象名称搜索
- 新增、编辑、删除成功后刷新当前页签列表

行权限列表字段沿用静态壳：

- `类型`
- `受限对象`
- `白名单`
- `操作`

### 6.2 对象类型

行权限支持以下对象类型：

- 角色
- 用户
- 系统变量

角色、用户来源：

- 使用真实角色、用户接口
- 支持按名称或账号搜索

系统变量来源：

- 使用真实系统变量接口或现有变量枚举接口
- 同一系统变量是否重复由后端判断

### 6.3 行权限条件

行权限条件字段来源：

- 当前选中数据集字段接口
- 只展示可参与过滤的字段
- 字段类型需要传给条件组件
- 复用现有 `RowAuth` 条件树组件

无可过滤字段时：

- 点击新增后提示当前数据集不可配置行权限
- 不打开无效弹窗

### 6.4 白名单

行权限支持白名单：

- 白名单作为规则的一部分保存
- 编辑规则时回显已选白名单
- 白名单用户来源使用真实用户接口
- 白名单用户支持按用户名称或账号搜索
- 白名单含义为：以上行权限规则对白名单用户不生效

## 7. 列权限设计

### 7.1 列表

列权限列表展示规则：

- 当前列表只展示当前选中数据集的列权限规则
- 默认按接口返回顺序展示
- 支持分页
- 支持对象类型筛选
- 支持受限对象名称搜索
- 新增、编辑、删除成功后刷新当前页签列表

列权限列表字段沿用静态壳：

- `类型`
- `受限对象`
- `白名单`
- `操作`

本期明确不增加外层 `字段规则摘要` 列。

字段规则摘要和规则预览只在列权限弹窗内部展示，弹窗内字段规则表包含：

- `字段名称`
- `规则预览`
- `操作`

### 7.2 对象类型

列权限支持以下对象类型：

- 角色
- 用户

列权限不支持系统变量对象类型。

### 7.3 字段规则

列权限字段来源：

- 当前选中数据集字段接口
- 只展示可配置列权限的字段

字段规则支持：

- 字段禁用
- 字段脱敏
- 内置脱敏规则
- 自定义脱敏规则

字段规则配置在列权限弹窗内部完成，不在外层列表直接编辑。

### 7.4 脱敏规则弹窗

脱敏规则弹窗是列权限弹窗内的子弹窗。

交互规则：

- 从列权限弹窗的字段规则行打开
- 在脱敏规则弹窗内选择或配置规则
- 点击确认后只回写列权限弹窗中的当前字段规则
- 最终保存由列权限弹窗确认按钮触发
- 关闭脱敏规则弹窗不影响列权限弹窗已填写的其他内容

### 7.5 白名单

列权限支持白名单：

- 白名单作为规则的一部分保存
- 编辑规则时回显已选白名单
- 白名单用户来源使用真实用户接口
- 白名单用户支持按用户名称或账号搜索
- 白名单含义为：以上列权限规则对白名单用户不生效

## 8. 保存、删除和重复规则

### 8.1 保存策略

本期采用弹窗级立即保存：

- 行权限弹窗点击确认后立即调用保存接口
- 列权限弹窗点击确认后立即调用保存接口
- 保存成功后关闭弹窗并刷新当前页签列表
- 保存失败时保留弹窗，并展示后端错误信息

本期不做页面级 `保存全部`。

### 8.2 删除策略

删除前必须二次确认。

行权限删除文案：

- 对象名存在：`确认删除【对象名】的行权限规则？删除后不可恢复。`
- 对象名缺失：`确认删除该行权限规则？删除后不可恢复。`

列权限删除文案：

- 对象名存在：`确认删除【对象名】的列权限规则？删除后不可恢复。`
- 对象名缺失：`确认删除该列权限规则？删除后不可恢复。`

删除成功后：

- 展示成功提示
- 刷新当前页签列表

### 8.3 重复规则

重复规则不由前端判断。

原因：

- 列表分页时前端无法可靠判断全部已有规则
- 不同对象类型和系统变量组合的冲突规则应以后端为准

前端允许提交，由后端判断并返回重复或冲突错误提示。

## 9. 加载和错误处理

### 9.1 加载策略

页面加载顺序：

1. 校验登录态和菜单权限点
2. 加载数据集树
3. 根据 query 或默认规则选中数据集
4. 默认加载行权限列表
5. 切换到列权限时再加载列权限列表

只加载当前页签数据，不提前加载另一个页签。

### 9.2 空状态

空状态沿用静态壳：

- 无规则时展示原型内的空状态
- 无数据集时保留页面空壳
- 不额外添加说明型大段文案

### 9.3 错误提示

接口错误统一使用 DataEase 现有消息提示组件。

展示优先级：

1. 后端返回的明确错误信息
2. 前端兜底错误信息

新增、编辑、删除成功需要展示成功消息。

## 10. 鉴权和组织上下文

### 10.1 菜单权限点

新增独立权限点：

- `行列权限配置`

只有具备该权限点的用户可以从系统设置左侧菜单看到并进入页面。

### 10.2 登录鉴权

正式路由必须走现有登录鉴权。

query 直接打开不代表跳过鉴权。

若用户未登录：

- 按现有系统逻辑跳转登录

若用户无权限点：

- 按现有系统逻辑隐藏菜单或阻止访问

### 10.3 组织上下文

页面的数据集、角色、用户、系统变量、白名单均基于当前组织上下文查询。

默认不跨组织配置。

## 11. 与其他模块关系

### 11.1 权限配置

`权限配置` 保持为系统权限配置能力：

- 按用户配置
- 按资源配置

本期不再在 `权限配置` 页内展示行权限、列权限页签。

### 11.2 问数配置

`问数配置` 本期不提供行列权限入口。

原因：

- 当前数据集操作菜单仍在开发中
- 本期避免行列权限与问数资源管理互相影响
- 后续可通过 query 机制接入快捷跳转

预留后续能力：

- 数据集行操作菜单点击后跳转 `/sys-setting/row-column-permission?datasetId=xxx&tab=row`
- 数据集行操作菜单点击后跳转 `/sys-setting/row-column-permission?datasetId=xxx&tab=column`

该能力本期不启用。

## 12. 验收标准

### 12.1 菜单和路由

- 系统设置左侧存在 `行列权限配置`
- `行列权限配置` 与 `权限配置`、`问数配置`、`用户管理`、`组织管理` 同级
- 进入路径为 `/sys-setting/row-column-permission`
- 权限配置页内不再展示行权限、列权限页签
- 问数配置数据集操作菜单不出现行权限、列权限入口

### 12.2 页面视觉

- 页面主体与 5 个已确认静态壳保持一致
- 高分辨率下页面宽度自适应，不出现大面积无效空白
- 表格列宽合理，不出现第一列和第二列异常大空隙
- 字体、密度、边界与系统设置统一规范一致

### 12.3 行权限

- 可选中数据集并加载行权限列表
- 支持角色、用户、系统变量三类对象
- 支持新增、编辑、删除
- 支持白名单
- 支持分页、筛选、搜索
- 删除前有二次确认
- 保存失败展示后端错误

### 12.4 列权限

- 可选中数据集并加载列权限列表
- 支持角色、用户两类对象
- 支持新增、编辑、删除
- 支持字段禁用和字段脱敏
- 支持内置和自定义脱敏规则
- 支持白名单
- 支持分页、筛选、搜索
- 外层列表不展示字段规则摘要列
- 字段规则预览只在列权限弹窗内展示

### 12.5 浏览器验证

浏览器自动化验证前必须按当前工程习惯重启对应服务：

- 修改 DataEase 前端代码后执行 `./dataease-web.sh`
- 修改 DataEase 后端代码后执行 `./dataease-app.sh`

每次 agent-browser 自动化测试前：

- 先 `close all`
- 再打开正式页面验证
- 所有验证截图保存到当前工程 `tmp/` 目录
