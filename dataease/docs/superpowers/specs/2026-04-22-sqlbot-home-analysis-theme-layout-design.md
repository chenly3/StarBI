# 智能问数首页分析主题布局设计

## 1. 背景

当前智能问数首页仍然沿用旧的顶部切换结构，核心问题有三类：

- 顶部 `问数 / 报告 / 搭建` 切换不符合目标原型表达。
- 首页主内容区没有前置“分析主题 / 数据集选择”入口，用户进入后缺少结构化的分析范围选择。
- 现有问答页虽然支持基于数据集/数据文件问数，但还缺少在“当前分析主题下命中多个数据集”时的多选澄清链路，这与 Quick BI 的分析主题问数体验不一致。

用户要求：

- 删除首页顶部原有的 `问数 / 报告 / 搭建` 切换。
- 参考 Calicat 原型，把首页改为“左侧导航 + 主内容区”布局。
- 主内容区上半部分替换为“分析主题数据集选择区”。
- 主内容区下半部分继续保留现有问答输入区、推荐问题等能力。
- 同一轮设计中补上“问答页命中当前分析主题下多个数据集时的卡片式澄清 UI”。
- 在改真实首页之前，必须先做一张**完整首页静态 Vue 母版页**，并先给用户截图确认。

## 2. 原型来源

本次首页布局改造参考以下两个 Calicat 原型：

1. 左侧导航菜单栏  
   `https://www.calicat.cn/design/2043250442777505792?node-id=0e162c76-e133-4e44-b9db-bcda0883bb82&node-type=group`
2. 分析主题数据集选择区  
   `https://www.calicat.cn/design/2043250442777505792?node-id=bd513881-da3f-4aec-abf6-2d25f65f9118&node-type=group`

## 3. 目标

本次要同时完成两部分设计：

1. **首页布局改造**
   - 左侧新增导航菜单栏
   - 主内容区上半部分新增分析主题数据集选择区
   - 下半部分保留现有问答输入与推荐问题

2. **问答页多数据集澄清**
   - 当当前上下文是 `全部` 或某一分析主题，且问题命中多个候选数据集时
   - 在聊天流中插入卡片式澄清 UI
   - 用户勾选 1 个或多个数据集后继续问数

## 4. 范围

### 4.1 在范围内

- 智能问数首页布局重排
- 完整首页静态 Vue 母版页
- 左侧导航菜单栏视觉与交互
- 分析主题数据集选择区
- 首页与现有问答区上下文联动
- 问答页多数据集澄清卡片式 UI

### 4.2 不在范围内

- 不把 `小星报告`、`小星搭建` 接成真实功能页
- 不新增报告页、搭建页、知识库页、权限页
- 不改其它系统设置页面
- 不做跨多个分析主题同时问数
- 不重写现有问答执行主逻辑，只在入口和澄清链路补齐

## 5. 交互模型

## 5.1 首页整体结构

首页最终结构仍由现有智能问数首页承载页负责渲染，但本次设计先定义一张**完整首页静态母版页**，它必须完整包含：

- 顶部蓝色全局导航栏
- 左侧三段结构
  - 智能问数产品卡
  - `小星问数 / 小星报告 / 小星搭建 / 历史对话` 菜单卡
  - 最近问数列表
- 右侧上半：分析主题数据集选择区
- 右侧下半：现有问答输入区与推荐问题区

也就是说，这次的“静态母版页”不是只画两个局部区块，而是整张首页母版。

## 5.2 左侧导航菜单栏

左侧菜单项固定为：

- `小星问数`
- `小星报告`
- `小星搭建`
- `历史对话`

行为约束：

- `小星问数` 默认激活，承载本期真实内容。
- `小星报告` 和 `小星搭建` 本期只做占位导航态：
  - 可点击
  - 可高亮
  - 不切真实页面
  - 不改变主内容区当前真实问数内容
- `历史对话` 继续复用现有历史对话能力，不重做历史系统。

静态母版页中：

- 四个菜单项都应完整出现
- 视觉上以 `小星问数` 作为激活项
- `小星报告 / 小星搭建` 先只做视觉导航态
- `历史对话` 在静态母版页里只做视觉占位，不接真实历史逻辑

## 5.3 分析主题数据集选择区

选择区核心模型如下：

- 顶部保留一个 `全部` 入口
- 后续按顺序展示分析主题标签
- 任一时刻只存在一个主题上下文：
  - `全部`
  - 某一个分析主题

主题切换规则：

- 默认进入首页时选中 `全部`
- 切到某个分析主题时，只展示该主题下的数据集
- 切回 `全部` 时，展示跨主题可用数据集

数据对象切换：

- 保留 `数据集 / 数据文件` 二选一结构
- 视觉上对齐原型
- 逻辑上继续复用现有数据集/数据文件模型

数据集卡片规则：

- 卡片展示名称和简要字段标签
- 支持选中态
- 选中后把当前问数上下文同步给下方问答区
- 如切换到新的主题，而原选中数据集不属于新主题，则自动清空原选择

静态母版页中，这一整块必须完整出现：

- 标题与副标题
- `全部 / 分析主题` tabs
- `数据集 / 数据文件 / 试用` 结构
- 多张数据集卡片
- `查看全部可用数据` 卡片

不能再用“预留区域/占位框”代替。

## 5.4 首页与问答区联动

下方现有问答区保留，但角色重新定义为“在已选范围内问数”：

- 默认状态：
  - 未显式选择具体数据集时，问答区仍可见
  - 引导文案改成：`直接提问，或先在上方选择分析主题 / 数据集`
- 只选了 `全部` 或某个分析主题：
  - 问答区进入“有主题上下文，无明确数据集”的状态
- 选了具体数据集：
  - 问答区进入“有主题上下文 + 有数据集上下文”的状态

静态母版页中，这一部分也必须完整出现为视觉母版：

- 问答输入卡
- 选择数据入口
- 已选数据标签
- 发送按钮
- 推荐问题区

历史恢复：

- 进入历史会话后，上方选择区同步恢复到该会话的主题/数据集上下文
- 不改变现有历史会话加载主逻辑

## 5.5 问答页多数据集澄清卡

### 触发条件

澄清卡只在以下条件同时满足时出现：

- 当前上下文是 `全部` 或某一个分析主题
- 用户问题命中多个候选数据集
- 当前尚未被收敛到单一数据集

### 交互形式

采用**聊天流中的卡片式澄清 UI**，不使用弹窗，不使用抽屉。

卡片包含：

- 一句澄清说明：例如“当前问题匹配到多个数据集，请选择后继续分析”
- 多个候选数据集条目
- 每个条目支持勾选
- `继续分析` 按钮

### 选择规则

- 允许勾选 1 个或多个数据集
- 点击 `继续分析` 后：
  - 将这些数据集作为当前问题的有效上下文
  - 继续触发问答执行

### 与首页选择区的关系

- 如果首页已经明确选中了某一个具体数据集，通常不再弹多数据集澄清卡
- 如果首页只选了 `全部` 或某个主题，而问题在该上下文内命中多个数据集，则弹卡

## 6. 与 Quick BI 对齐点

本设计与 Quick BI 的分析主题问数模型保持一致：

- 支持 `全部` 入口
- 一次只处于一个主题上下文
- 在该上下文下可进一步选择数据集
- 命中多个数据集时，通过卡片式澄清继续收敛
- 不做“同时选多个分析主题”的复杂交互

## 7.1 先产出本地静态 Vue 原型参考页

用户补充要求：

- 不要直接在真实首页里边看图边改。
- 先把 Calicat 对应的首页原型区块翻译成一份**本地静态 Vue 参考页**。
- 启动项目后打开这份静态参考页，先截图给用户确认视觉和结构可行，再继续真实首页改造。

因此本次实施顺序必须调整为：

1. **静态参考页阶段**
   - 先用 Calicat 原型还原出一份本地静态 Vue 页面
   - 内容是**完整首页静态母版**
   - 至少完整包含：
     - 顶部蓝色全局导航栏
     - 左侧导航菜单栏
     - 最近问数列表
     - 分析主题数据集选择区
     - 下方问答输入卡与推荐问题区
   - 不接真实数据
   - 不接真实问答逻辑
   - 不挂正式业务菜单

2. **静态页确认阶段**
   - 启动项目
   - 打开静态参考页
   - 截图给用户确认

3. **真实首页接入阶段**
   - 再以静态参考页为视觉母版
   - 改造真实首页
   - 把选择区和问答区联动接回现有逻辑

## 7.2 静态参考页约束

静态参考页的定位是“视觉母版”，不是正式业务页：

- 只做原型对应的 Vue 结构和样式
- 不接真实主题数据、不接真实数据集列表
- 不承载首页真实路由入口
- 不替代真实智能问数首页
- 只用于：
  - 视觉确认
  - 代码参考
  - 后续真实首页还原时的结构蓝本

## 8. 设计原则

- 视觉上尽量贴近 Calicat 原型
- 逻辑上尽量复用现有问答、历史、选择链路
- 首页只做入口重排和上下文前置，不重写问答主引擎
- 报告/搭建只做导航态，不扩范围
- 澄清卡走聊天流内嵌卡片，不增加额外弹层复杂度
- 先静态参考页、后真实接入，避免边看截图边猜布局

## 9. 风险

- 现有首页文件可能已经较重，直接堆新结构会增加维护难度，因此实现阶段建议适度拆组件
- 首页选择区与历史恢复、问答上下文同步之间需要谨慎处理，避免出现主题/数据集状态不一致
- 若现有问答逻辑默认强依赖“已选单一数据集”，则多数据集澄清卡需要在调用链上补一层上下文适配

## 10. 本次结论

本次首页改造不只是“换个布局”，而是补齐 Quick BI 风格的分析主题问数闭环：

- 先输出一份**完整首页静态 Vue 原型参考页**
- 用户确认静态参考页视觉可行后
- 首页先选 `全部 / 分析主题 / 数据集`
- 下方继续问
- 命中多个数据集时，再通过聊天流卡片澄清后继续分析
