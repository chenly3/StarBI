# 用户管理 Remote Plugin First 分析拆单卡

## 1. 文档目标

- 本文档不是 `writing-plans` 级别的实现计划。
- 本文档只回答一个问题：在 `remote/plugin-first` 前提下，用户管理模块现在可以先拆哪些分析任务，哪些必须继续阻塞。
- 输出对象是产品、设计、前端宿主、远端 X-Pack 模块、后端合同联调 5 类协作方。

## 2. 当前冻结前提

### 2.1 已冻结

- 前端承载方式：`remote/plugin-first`
- 新增用户组织归属：保持“切换组织后新增”，不扩 `orgId`
- 外部用户：本期不做完整独立流转，但必须保留同入口扩展能力
- 自定义角色 `desc`：本期不阻塞主链路

### 2.2 未冻结

- `beforeUnmountInfo` 数值映射
- 重置密码是否展示默认密码

### 2.3 直接约束

- 当前仓库不进入本地 `/system/user` Vue 页面搭建。
- 下一轮动作以“远端插件模块拆解 + 宿主接管点确认 + 合同/原型补齐”为主。
- 任何依赖未冻结协议的链路，只能做分析或联调约定，不能进入最终编码。

## 3. 拆单视角

| 视角 | 关注点 |
| --- | --- |
| 宿主前端 | 路由、插件壳、入口、权限激活、接口包装缺口 |
| 远端插件前端 | 用户列表、表单、弹窗、导入、角色链路 |
| 后端合同 | DTO/VO、接口语义、联调映射 |
| 产品/原型 | 当前组织提示、外部用户入口、风险提示 |
| 测试验收 | 当前组织隔离、导入部分成功、角色影响提示 |

## 4. 可切分析卡

### A1. 宿主侧插件接管点梳理

**目标**

- 明确当前仓库在用户管理模块里到底负责什么，不负责什么。

**范围**

- `/system` 入口
- `plugin` 路由改写
- `components/plugin` 壳层
- 远端 `DEXPackTs.umd.js` 注入链路

**当前依据**

- `core/core-frontend/src/router/establish.ts`
- `core/core-frontend/src/components/plugin/src/index.vue`
- `core/core-frontend/src/components/plugin/src/ImportXpackTool.ts`
- `core/core-frontend/src/layout/components/SystemCfg.vue`
- `core/core-frontend/src/layout/components/Header.vue`

**输出**

- 一张“宿主负责 / 远端负责 / 当前仓库缺口”的边界表

**验收口径**

- 能回答“为什么本轮不能直接建本地用户管理页”
- 能回答“宿主侧是否还需要补 API wrapper 或权限入口”

### A2. 用户列表与查询链路拆解

**目标**

- 冻结用户列表主链路的远端实现边界。

**范围**

- 列表字段
- 搜索
- 筛选：来源、启用状态、角色
- 分页
- 批量选择

**当前依据**

- 官方文档 `1.1`、`1.7`
- Calicat 用户列表节点 `8a93fae7-5861-4018-b125-aef71633f858`
- `UserGridRequest`
- `UserGridVO`
- `core/core-frontend/src/api/user.ts`

**明确结论**

- 当前列表按“当前组织视角”实现
- `所属组织` 仍然不是现有合同字段，不能当作一期开发表达

**输出**

- 列表字段矩阵：`现有合同字段 / 原型字段 / 一期可交付字段 / 后置字段`

**验收口径**

- 开发不会再把 `所属组织` 当成已存在字段
- 列表筛选口径与 `UserGridRequest` 一致

### A3. 新增与编辑用户链路拆解

**目标**

- 在不改合同的前提下冻结新增/编辑用户的交互边界。

**范围**

- 新增用户
- 编辑用户
- 当前组织提示
- 角色多选
- 启用状态

**当前依据**

- 官方文档 `1.2`、`1.4`
- Calicat 添加用户节点 `8b26da3d-6193-42af-937d-ae47cbb2d7b3`
- `UserCreator`
- `UserEditor`
- `userCreateApi`
- `userEditApi`

**明确结论**

- 不扩 `orgId`
- 新增必须显式提示“当前组织”
- 账号创建后不可改

**输出**

- 新增/编辑共用弹窗约束清单

**验收口径**

- 产品、设计、前端对“切组织后新增”的口径一致
- 不再讨论显式组织字段作为一期范围

### A4. 删除、启停、重置密码链路拆解

**目标**

- 把用户列表行级操作中可立即推进的部分和待决项分开。

**范围**

- 单删
- 批量删
- 启停
- 重置密码

**当前依据**

- 官方文档 `1.5`、`1.6`
- `userDelApi`
- `batchDelApi`
- `switchEnableApi`
- `defaultPwdApi`
- `resetPwdApi`

**明确结论**

- 删除、批量删除、启停可以先行分析拆解
- 重置密码只能先收敛为“恢复为初始密码”的主链路
- 是否展示默认密码仍是编码前 gate

**输出**

- 行级操作状态表：`已冻结 / 待确认 / 可直接实现`

**验收口径**

- 团队不会误把“默认密码展示”当成已定需求

### A5. 批量导入与导入结果链路拆解

**目标**

- 冻结导入主链路与结果反馈链路。

**范围**

- 模板下载
- 文件上传
- 导入
- 部分成功
- 错误报告下载
- 返回用户列表 / 继续导入

**当前依据**

- 官方文档 `1.3`
- Calicat 节点：
  - 批量导入 `4b7c7f7c-1ded-46c7-a4a9-98c6376efc53`
  - 导入结果 `5a55647b-3c4c-4ba9-a6ba-cf60c0ff6e3a`
- `downExcelTemplateApi`
- `importUserApi`
- `downErrorRecordApi`
- `UserImportVO`

**明确结论**

- 一期必须支持部分成功
- 错误报告下载属于主链路，不是增强项

**输出**

- 导入状态流转表

**验收口径**

- 导入失败不会只停留在 toast，而有结果页/结果弹窗承接

### A6. 角色列表与添加组织用户链路拆解

**目标**

- 在外部用户后置前提下，先收敛角色列表与添加组织用户主链路。

**范围**

- 角色列表
- 角色详情
- 已绑用户
- 添加组织用户双栏选择

**当前依据**

- 官方文档 `2.1`、`2.2`
- Calicat 节点：
  - 角色列表 `cc063f54-e780-4229-9622-595abd9bd909`
  - 添加组织用户 `eb5a063f-6da2-4b6a-a3db-d7bdd91c73d7`
- `searchRoleApi`
- `roleDetailApi`
- `mountUserApi`
- `userOptionForRoleApi`
- `userSelectedForRoleApi`

**明确结论**

- 本期先保证“组织用户加人”闭环
- 外部用户只保留扩展位，不进入最终实现拆单

**输出**

- 角色页信息架构图

**验收口径**

- 角色页主链路不依赖外部用户原型即可闭环

### A7. 创建自定义角色链路拆解

**目标**

- 冻结一期“创建自定义角色”的最小闭环。

**范围**

- 角色名称
- 继承类型单选
- 创建成功

**当前依据**

- 官方文档 `2.4`
- Calicat 节点 `ee600d04-23ff-4a76-bb7f-5e5c417461ed`
- `RoleCreator`
- `roleCreateApi`

**明确结论**

- 一期主链路只要求 `name + typeCode`
- `desc` 后置，不阻塞主闭环

**输出**

- `typeCode` 与“普通用户/组织管理员”映射表

**验收口径**

- 创建链路不会因为 `desc` 是否展示继续卡住

## 5. Blocked 分析卡

### B1. 角色移除用户最终交互

**阻塞原因**

- `beforeUnmountInfo` 只返回 `Integer`，数值语义未冻结。

**当前能做**

- 保留文案、风险分级、待确认映射草案

**当前不能做**

- 最终交互实现拆单
- 前端分支逻辑编码
- 联调用例最终断言

**解锁条件**

- 前后端确认 `Integer -> 文案/行为` 映射表

### B2. 外部用户完整流转

**阻塞原因**

- Calicat 尚未给出外部用户完整原型
- 当前只确认“保留扩展位”，未确认最小可验收交付
- 搜索方式、空态提示、提交成功后的回流方式均未冻结

**当前能做**

- 保留入口语义
- 标注将来需要 `searchExternalUser` + `mountExternalUser`

**当前不能做**

- 远端插件最终交互实现拆单
- 精确搜索交互与空态设计

**解锁条件**

- 补原型
- 补最小验收口径
- 冻结搜索/空态边界

### B3. 重置密码展示默认密码

**阻塞原因**

- 仍未冻结是否展示默认密码明文

**当前能做**

- 主链路先按“恢复为初始密码”分析

**当前不能做**

- 明文展示交互拆单
- 复制按钮、安全提示、审计策略的最终实现拆单

**解锁条件**

- 产品确认展示策略

## 6. 协作清单

| 协作方 | 需要给出的确认 |
| --- | --- |
| 产品 | 重置密码展示策略；外部用户本期最小交付；角色移除提示口径 |
| 设计 | 当前组织提示；外部用户入口占位；风险提示弹窗表达 |
| 前端宿主 | 插件入口、路由、壳层是否仍需补接管逻辑 |
| 远端插件前端 | 主链路页面与弹窗拆解边界 |
| 后端合同方 | `beforeUnmountInfo` 数值语义；如需默认密码展示是否有附加安全约束 |

## 7. 下一步动作建议

1. 先基于本文件完成一次“remote/plugin-first 拆单评审”。
2. 评审后，把 A1-A7 转成研发输入整理项。
3. 对 B1-B3 单独挂阻塞卡，不混入主链路工期估算。

## 8. 结论

- 用户管理模块当前已经可以进入 `remote/plugin-first` 的分析拆单阶段。
- 但这仍然不是编码计划，也不是本地页面开发计划。
- 下一阶段的正确动作是：先拆清宿主与远端边界、主链路与 blocked 链路，再决定是否进入正式 `write plan`。
