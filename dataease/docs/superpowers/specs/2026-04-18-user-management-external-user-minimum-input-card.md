# 用户管理外部用户最小输入卡

## 1. 文档目标

- 本文档只服务于用户管理模块中“角色添加用户”的外部用户分支。
- 它不是实现计划，也不是最终原型稿。
- 它的作用是补齐一个最小评审输入，避免外部用户链路长期停留在“知道有接口，但不知道最小交付长什么样”。

## 2. 当前边界

### 2.1 已确认

- 官方文档存在“组织用户 + 外部用户”双流转语义。
- 当前仓库已有合同：
  - `GET /role/searchExternalUser/{keyword}`
  - `POST /role/mountExternalUser`
- `searchExternalUser` 返回单个 `ExternalUserVO`
- `mountExternalUser` 只要求 `rid + uid`

### 2.2 未确认

- Calicat 尚未提供独立外部用户弹窗原型
- 搜索词到底允许“账号 / 用户 ID / 统一关键字”还未在产品侧冻结
- 空态、失败态、成功回流方式未冻结

### 2.3 当前建议

- 外部用户不进入一期主实现
- 先补“最小输入卡”，作为后续原型补页和评审输入

## 3. 最小交互建议

### 3.1 入口语义

- 入口仍然挂在“角色详情 -> 添加用户”同一弹窗内
- 不额外拆成独立页面
- 入口形态建议：
  - 组织用户
  - 外部用户

### 3.2 搜索模式

- 建议采用“单输入框 + 精确查找单人”模式
- 不建议一期扩成模糊列表搜索

**原因**

- 当前 `searchExternalUserApi` 只有一个 `keyword` 入参
- 当前 `ExternalUserVO` 返回单个对象，不是分页列表
- 现有合同更适合“查到一个人 -> 确认挂载”

### 3.3 搜索结果展示

- 搜索命中后，只展示单条卡片或单行结果
- 建议字段：
  - 姓名
  - 账号
  - 邮箱
  - 手机

**原因**

- `ExternalUserVO` 只提供这 5 个字段：
  - `uid`
  - `account`
  - `name`
  - `email`
  - `phone`

### 3.4 建议提交流程

1. 选择目标角色。
2. 打开“添加用户”弹窗。
3. 切到“外部用户”入口。
4. 输入精确搜索词。
5. 若命中单个用户，展示用户信息卡。
6. 点击“添加到当前角色”。
7. 调用 `mountExternalUser(rid, uid)`。
8. 成功后返回当前角色成员列表并刷新。

## 4. 建议最小验收口径

### 4.1 评审建议包含

- 有外部用户入口语义
- 有单输入框搜索
- 有命中单人后的结果承接
- 有“添加到当前角色”提交动作
- 提交成功后回到当前角色成员列表

### 4.2 本轮不建议纳入

- 分页搜索结果
- 模糊搜索推荐列表
- 多用户批量外部挂载
- 复杂筛选条件
- 独立外部用户页面

## 5. 状态定义

### 5.1 初始态

- 展示搜索输入框
- 展示提示文案：
  - 请输入账号或用户 ID 精确搜索

### 5.2 命中态

- 展示单个用户结果卡
- 显示“添加到当前角色”按钮

### 5.3 无结果态

- 文案建议：
  - 暂无匹配用户，请检查账号或用户 ID 是否正确

### 5.4 失败态

- 文案建议：
  - 查询失败，请稍后重试

### 5.5 成功态

- 文案建议：
  - 添加成功
- 行为：
  - 建议关闭当前外部用户视图
  - 建议刷新当前角色成员列表

## 6. 待确认项

| 主题 | 当前建议 | 待谁确认 |
| --- | --- | --- |
| 搜索词口径 | 账号或用户 ID 精确搜索 | 产品 / 后端合同方 |
| 入口样式 | 与组织用户同弹窗双模式切换 | 设计 |
| 空态文案 | 提示检查账号或用户 ID | 产品 |
| 成功回流 | 返回当前角色成员列表并刷新 | 产品 / 前端 |

## 7. 与当前拆单的关系

- 这张卡不改变当前主结论：外部用户完整流转仍然 blocked。
- 它只是把 blocked 项从“概念占位”推进到“最小输入已定义”。
- 后续如果要真正进入实现，还需要：
  - Calicat 原型补页
  - 搜索词口径冻结
  - 空态与成功回流冻结

## 8. 结论

- 基于当前合同，外部用户最小可行形态更适合“单人精确搜索 + 单次挂载”。
- 这能与现有 `ExternalUserVO` 和 `MountExternalUserRequest` 对齐，也不会误导团队提前设计成复杂列表流。
