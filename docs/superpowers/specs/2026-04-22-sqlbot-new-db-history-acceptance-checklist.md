# SQLBot New 数据库持久化历史验收清单

## 1. 目标

用于人工验收 `sqlbot-new` 的历史真源已经从浏览器缓存切换到 SQLBot 数据库，并验证数据集问数、数据文件问数、多数据源切换、历史恢复、继续追问都能正常工作。

## 2. 验收前置

- SQLBot 后端已启动
- DataEase 前端已启动
- 数据库已执行到包含 `chat_session_snapshot` 与 `chat_session_event` 的最新 Alembic 版本
- 使用 `admin / DataEase@123456` 登录本地环境

## 3. 数据集问数验收

### 3.1 新建数据集问数历史

1. 进入 `sqlbot-new`
2. 选择数据集 `茶饮订单明细`
3. 提问：`按门店统计销售金额前5名`
4. 预期：
   - 结果页正常返回
   - 左侧 `最近问数` 出现 1 条历史
   - 标题为当前选择数据的标题

### 3.2 多数据源切换仍为同一条历史

1. 在已有结果页中切换到 `茶饮原料费用`
2. 预期：
   - 结果区出现“上下文切换”卡片
   - 左侧仍只有同一条历史，不新增第二条
   - 输入区当前选择数据更新为 `茶饮原料费用`

### 3.3 数据集历史恢复

1. 回到首页
2. 展开左侧 `历史对话`
3. 点击刚才那条历史
4. 预期：
   - 原有问答内容恢复
   - “上下文切换”卡片恢复
   - 输入区当前选择数据恢复为最后激活的数据集

### 3.4 恢复后继续追问

1. 在恢复后的结果页继续提问：`按门店统计原料费用前5名`
2. 预期：
   - 新问题作为下一轮问答追加
   - 沿用 `茶饮原料费用` 作为当前上下文
   - 左侧仍为同一条历史

## 4. 数据文件问数验收

### 4.1 新建数据文件问数历史

1. 进入 `sqlbot-new`
2. 打开“选择数据”
3. 切换到 `数据文件`
4. 选择 `StarBI文件问数演示`
5. 提问：`按月份统计 sales_amount 前5名`
6. 预期：
   - 结果页正常返回
   - 左侧 `最近问数` 出现 1 条文件历史
   - 副标题为 `Excel 文件`

### 4.2 文件问数恢复

1. 回到首页
2. 展开左侧 `历史对话`
3. 点击 `StarBI文件问数演示`
4. 预期：
   - 问答内容恢复
   - 输入区文件标签恢复为 `StarBI文件问数演示`
   - 指标/维度标签仍为文件字段

### 4.3 文件恢复后继续追问

1. 在恢复后的结果页继续提问：`按区域统计 profit 前3名`
2. 预期：
   - 新问题作为下一轮问答追加
   - 仍沿用 `StarBI文件问数演示`
   - 不要求重新选择文件

## 5. 清缓存验收

### 5.1 清浏览器本地缓存后恢复

1. 先完成一条数据文件问数历史
2. 清空浏览器 `localStorage/sessionStorage`
3. 重新登录 `sqlbot-new`
4. 预期：
   - 文件上下文或文件历史仍能从后端恢复
   - 不依赖浏览器缓存存在

## 6. 数据库验收

### 6.1 快照表

检查 `chat_session_snapshot`：

- 每条新 `sqlbot-new` 会话有 1 条 snapshot
- `active_source_kind` 正确
- `selection_title / selection_meta` 正确
- `latest_record_id / latest_question` 会随问答推进更新

### 6.2 事件表

检查 `chat_session_event`：

- 新会话会写 `session_init`
- 数据源切换会写 `context_switch`
- 同一会话的事件按 `create_time` 递增

## 7. 失败判定

任一情况出现都判定为未通过：

- 左侧历史为空，但当前问答已完成
- 多数据源切换后左侧新增第二条历史
- 历史恢复后输入区当前选择数据丢失
- 文件历史恢复后文件标签丢失
- 清浏览器缓存后历史完全丢失
- 数据库没有对应 snapshot/event 记录

## 8. 当前实现范围

- 只覆盖 `sqlbot-new`
- 不迁移旧浏览器缓存历史
- 老 `sqlbot` 页面不在本轮验收范围内
