# Query Resource Learning Design

> Scope: StarBI `系统设置 -> 问数配置 -> 问数资源管理` 中的“学习能力”完整设计。

## Goal

把 `DataEase` 数据集从“可查询资源”升级为“可问、可懂、可推荐、可持续优化”的问数资源，并建立从学习、问数、反馈到二次学习的闭环。

## Product Positioning

“学习”不是一个单一按钮动作，而是一条面向智能问数的资源生产线。

普通 BI 数据集解决的是：

- 系统能不能查这个数据

问数学习解决的是：

- 用户用自然语言问的时候，系统能不能理解并答对

因此，学习能力是 `问数资源管理` 的核心，不是附属配置项。

## Core Definitions

### 1. Query Resource

问数资源是已纳入智能问数范围、并具备学习结果的数据集对象。

### 2. Learning

学习是对数据集的结构、字段、语义、维值、术语、SQL 示例和问数表现进行加工，生成一份可供问数引擎直接使用的语义资产。

### 3. Learning Result

学习结果是结构化沉淀物，用于：

- 数据集召回
- 字段召回
- 维值匹配
- 推荐问题生成
- SQL 生成约束
- 问数结果解释
- 学习质量评估

## Learning Outputs

每次学习应至少产出以下结果：

| 输出项 | 说明 | 用途 |
|---|---|---|
| 数据集基础画像 | 名称、描述、工作空间、修改时间、负责人 | 问数资源展示与基础上下文 |
| 字段语义画像 | 字段名、别名、类型、描述 | 字段理解与召回 |
| 字段角色分类 | 时间字段、维度字段、指标字段、敏感字段、ID 字段 | SQL 生成与问数控制 |
| 维值样本 | 地市、产品、渠道等高频值 | 自然语言过滤值匹配 |
| 推荐问题 | 自动生成的快捷提问 | 降低使用门槛 |
| 术语映射 | 业务词和字段/维值映射关系 | 提高语义命中率 |
| SQL 示例关联 | 高质量示例 SQL 和问题映射 | 约束模型生成 |
| 学习质量评分 | A/B/C/D 或数值评分 | 判断是否适合问数 |
| 学习任务记录 | 开始时间、结束时间、耗时、失败原因 | 后台运营和排障 |
| 回流优化记录 | 点踩、失败问题、人工修正结果 | 二次学习依据 |

## Product Flow

### Full Closed Loop

```text
数据集创建/更新
  -> 开启问数
  -> 触发学习任务
  -> 抽取元数据
  -> 抽取维值样本
  -> 生成语义与推荐问题
  -> 关联术语与 SQL 示例
  -> 质量评分
  -> 学习成功/失败
  -> 用户问数
  -> 记录反馈
  -> 自动/手动重学
```

### Learning Main Stages

| 阶段 | 动作 | 输出 |
|---|---|---|
| 1 | 资源登记 | 资源状态从“未开启”进入“待学习” |
| 2 | 元数据抽取 | 字段、类型、描述、关系、权限 |
| 3 | 语义分类 | 时间/维度/指标/敏感字段分类 |
| 4 | 样本抽取 | 高频维值、样例数据 |
| 5 | 问数资产生成 | 推荐问题、问数说明、提示上下文 |
| 6 | 知识联动 | 术语、SQL 示例、Prompt 绑定 |
| 7 | 质量评估 | 问数适配度和风险评分 |
| 8 | 状态更新 | 成功/失败/需重学 |
| 9 | 问数反馈采集 | 点赞点踩、失败问题、人工修正 |
| 10 | 回流学习 | 自动重学或人工重学 |

## Learning State Machine

| 状态 | 含义 | 允许动作 |
|---|---|---|
| 未开启 | 数据集未加入问数范围 | 开启问数 |
| 待学习 | 已开启但还没有学习结果 | 立即学习 |
| 学习中 | 学习任务正在执行 | 查看任务详情 |
| 学习成功 | 学习结果可用于问数 | 可问、可重学、可评分 |
| 学习失败 | 学习中断或结果不可用 | 查看失败原因、重试 |
| 需重新学习 | 资源已变更或质量下降 | 重新学习 |
| 已停用 | 从问数资源中移除 | 重新开启问数 |

## Failure Reason Taxonomy

学习失败必须给出可读的失败分类：

| 分类 | 示例 |
|---|---|
| 数据源问题 | 连接失败、超时、权限不足 |
| 数据集问题 | 字段缺失、无可用字段、结构异常 |
| 语义问题 | 字段描述缺失、分类失败、推荐问题生成失败 |
| 权限问题 | 数据集不可见、行列权限无法正确解析 |
| 模型问题 | 模型调用失败、生成结果为空 |
| 系统问题 | 存储失败、任务中断、索引失败 |

## Learning Quality Scoring

质量评分用于评估数据集是否适合进入问数范围。

建议拆成以下维度：

| 维度 | 判断点 |
|---|---|
| 字段完备度 | 描述、别名、类型是否齐全 |
| 结构可理解度 | 时间/维度/指标是否能明确识别 |
| 维值可用度 | 高频维值是否足够支撑问数 |
| 术语覆盖度 | 业务术语是否覆盖主要问题表达 |
| SQL 校准度 | 是否存在可复用 SQL 示例 |
| 权限稳定性 | 行列权限是否能稳定裁剪 |
| 问数表现 | 成功率、失败率、点踩率 |

建议输出：

- A：强推荐问数
- B：可问
- C：可问但建议补语义
- D：不建议开放问数

## Resource Management Page Responsibilities

“问数资源管理”页应成为学习能力的主运营入口。

### Must-have Fields

| 字段 | 说明 |
|---|---|
| 名称 | 数据集问数别名或资源名称 |
| 创建者 | 资源责任人 |
| 分析主题 | 所属主题或未归类 |
| 最后学习时间 | 最近学习完成时间 |
| 学习状态 | 成功/失败/学习中/需重学 |
| 质量评分 | A/B/C/D |
| 失败原因摘要 | 最近一次失败原因 |
| 推荐问题数量 | 当前已生成问题数 |

### Must-have Actions

| 动作 | 目的 |
|---|---|
| 学习数据 | 启动或重试学习 |
| 快捷提问设置 | 配置推荐问题 |
| 移除 | 从问数资源中下线 |

### Must-have Detail Views

| 详情页/抽屉 | 作用 |
|---|---|
| 预览数据 | 看样例数据和字段 |
| 学习任务详情 | 看过程、耗时、失败原因 |
| 质量详情 | 看评分和建议 |
| 推荐问题维护 | 看系统推荐和人工补充 |

## Permission Principles

学习能力必须继承 DataEase/X-Pack 权限体系。

### Rules

- 没有资源权限的数据集不能开启问数
- 行列权限必须参与问数结果裁剪
- 学习结果可基于全量元数据生成，但问数时必须按当前用户权限裁剪
- 学习操作权限和问数使用权限必须区分

### Suggested Permission Roles

| 权限 | 说明 |
|---|---|
| 集中管理权限 | 管理所有问数资源和学习任务 |
| 资源配置权限 | 管理自己负责的数据集问数配置 |
| 问数使用权限 | 仅使用问数，不可管理 |

## Automatic Re-learning Triggers

| 触发事件 | 是否自动重学 |
|---|---|
| 字段新增/删除/改名 | 是 |
| 字段描述变化 | 是 |
| 数据集 SQL 变化 | 是 |
| 数据源结构变化 | 是 |
| 术语库更新 | 可选 |
| SQL 示例更新 | 可选 |
| 点踩率上升 | 是 |
| 失败问题集中出现 | 是 |

## Frontend / Backend Boundaries

### Frontend Responsibilities

- 展示学习状态、任务、评分、失败原因
- 触发学习、批量学习、重新学习
- 维护推荐问题
- 查看学习详情和质量建议

### Backend Responsibilities

- 执行学习任务
- 抽取元数据和维值样本
- 生成推荐问题与问数上下文
- 关联术语和 SQL 示例
- 输出评分和失败原因
- 接收反馈并回流学习

## Backend Service Decomposition

| 服务 | 责任 |
|---|---|
| LearningTaskScheduler | 调度、重试、并发控制 |
| MetadataExtractor | 读取字段、关系、描述、权限 |
| ValueSampler | 抽取维值和样例数据 |
| SemanticGenerator | 字段分类、推荐问题、问数上下文 |
| KnowledgeBinder | 绑定术语、SQL 示例、Prompt |
| LearningScorer | 输出质量评分 |
| LearningFeedbackService | 点踩、失败问题、人工修正回流 |
| LearningResultStore | 保存学习版本和结果 |

## First-phase Delivery Suggestion

虽然是“完整学习”方案，但第一阶段可以按闭环优先落地，不要求一次性做到强智能。

### First-phase Must-do

- 元数据抽取
- 字段语义分类
- 维值样本抽取
- 推荐问题生成
- 术语/SQL 示例联动
- 学习状态机
- 学习任务详情
- 失败原因分类
- 质量评分
- 自动重学规则
- 点踩/失败问题回流入口

### Later-strengthening

- 智能优化建议更精准
- 质量评分模型更精细
- 回流学习更自动化
- 学习效果自适应优化

## Conclusion

完整学习能力不是“多一个学习按钮”，而是一条从 `数据集 -> 问数资产 -> 问数表现 -> 反馈回流 -> 二次学习` 的持续运营闭环。

第一阶段就应该把这条闭环搭起来，只是自动化和智能化程度可以分步增强。
